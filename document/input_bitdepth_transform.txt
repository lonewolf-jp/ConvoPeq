ビット深度変換(16bit/24bit/32bit→64bit double)シグナルフローの統合（設計図）
究極仕様：High-Fidelity Audio Input Pipeline v2.0
この実装では、ブロック間の連続性を維持しつつ、AVX2とMKLの性能を最高レベルで引き出します。
cpp
/*==============================================================================
    AudioInputPipeline_v2.cpp - 究極の連続性保護モデル
    - 16/24/32bit -> 64bit Double 変換
    - IIR型 DC-Blocking Filter (位相歪みを最小化した超低域カット)
    - ゲイン・スムージングによるヘッドルーム確保
==============================================================================*/

#include <immintrin.h>
#include <mkl.h>

class HighEndInputPipeline {
public:
    HighEndInputPipeline() {
        vmlSetMode(VML_HA | VML_FTZDAZ_ON);
        reset();
    }

    void reset() {
        dcStateL = 0.0; dcStateR = 0.0;
        lastGain = 0.988553; // -0.1dB
    }

    /**
     * @param src          入力（voidポインタ）
     * @param dst          64bit Double 出力（64byte Align済み）
     * @param n            サンプル数
     * @param bitDepth     16, 24, 32
     */
    void process(const void* src, double* dst, int n, int bitDepth) noexcept {
        // --- Step 1: 変換 (AVX2 高速パス) ---
        performConversion(src, dst, n, bitDepth);

        // --- Step 2: 正規化スケーリング (MKL VML) ---
        // 正規化係数と -0.1dB マージンを一括適用
        const double normFactor = getNormalizationFactor(bitDepth) * 0.988553;
        vdMul(n, dst, &normFactor, dst);

        // --- Step 3: [重要] 連続性を保つ DC-Blocking (IIR) ---
        // ブロックごとの平均値減算ではなく、フィルタリングで除去
        applyDCBlocker(dst, n);

        // --- Step 4: [重要] MKL VML による異常値のソフト・サニタイズ ---
        // 信号を -1.0 ～ 1.0 にクランプ（クリッピングではなく安全策）
        const double limit = 1.0;
        vdThreshold(n, dst, &limit, &limit, dst); // 正負両端のチェックが必要
    }

private:
    // IIRフィルタによるDCカット (0.5Hz @ 768kHz 相当)
    // ブロックを跨いで状態(dcState)を保持するため、ノイズが発生しない
    void applyDCBlocker(double* data, int n) noexcept {
        const double R = 0.999999; // 時定数 (768kHzに合わせて調整)
        for (int i = 0; i < n; ++i) {
            double x = data[i];
            double y = x - lastInput + R * dcStateL;
            lastInput = x;
            dcStateL = y;
            data[i] = y;
        }
    }

    // ビット深度ごとの AVX2 変換 (前述の実装をブラッシュアップ)
    void performConversion(const void* src, double* dst, int n, int bitDepth) {
        // [前述の AVX2 実装をここに統合。24bitは特に丁寧に符号拡張を行う]
    }

    double getNormalizationFactor(int bd) {
        if (bd == 16) return 1.0 / 32768.0;
        if (bd == 24) return 1.0 / 8388608.0;
        return 1.0 / 2147483648.0;
    }

    double dcStateL = 0.0, dcStateR = 0.0;
    double lastInput = 0.0;
    double lastGain = 1.0;
};
コードは注意してご使用ください。

ノイズを消し去るための実装アドバイス
IIR DC Blocker の採用:
以前の vdMean を使った手法は、ブロックの境界で位相がジャンプするため、「ジリジリ」としたノイズの最大の原因になります。この IIR 方式なら、768kHz の滑らかさを維持したまま DC だけを消せます。
24bit ロードの厳密化:
24bit 整数は 3 バイト整列のため、ロード時に隣のサンプルのビットが混入しやすいです。src[i*3] 方式の符号拡張を、MSVC Intrinsic の _mm_blend_epi8 等を使って厳密にガードしてください。
MKL VML 動作モードの再確認:
vmlSetMode(VML_HA) が「実際に計算を行っているスレッド」内で呼ばれているか、再度確認してください。
オーバーサンプリングとの整合性:
もしノイズが高域（高音）が流れる時だけ出るなら、それは「エイリアシング」です。OS の Kaiser-HB フィルタの係数が 64bit 精度で正しく計算されているか、Matlab や Octave の設計値と 15 桁以上一致しているか確認してください。
