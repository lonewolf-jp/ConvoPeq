1. アルゴリズムの核心：ハイブリッド構造
Non-uniform方式では、インパルス応答（IR）を「時間経過とともに大きくなるブロック」に分割します。
Head部 (低レイテンシ): 最初の数ミリ秒。ごく小さいブロックサイズ（例：64〜256サンプル）で、時間領域（Direct Form）または小さなFFTで処理。
Body部 (中レイテンシ): 中間のブロックサイズ（例：1024〜4096サンプル）。
Tail部 (高スループット): 巨大なブロックサイズ（例：16384〜65536サンプル）。バックグラウンドスレッドで計算。
2. MKL DFTI を用いた実装設計図
MKLは複数のFFT記述子（DFTI_DESCRIPTOR_HANDLE）を同時に保持できるため、サイズごとの「プラン」を事前に作成します。
① IRの分割とプリコンパイル (prepareToPlay)
cpp
// IRを異なるサイズのブロックに分割して周波数ドメインへ変換しておく
struct Partition {
    size_t fftSize;
    DFTI_DESCRIPTOR_HANDLE handle;
    std::vector<MKL_Complex16*> blocks; // 各セグメントの周波数データ
};

// 例: Head (1024), Body (4096), Tail (16384) の3階層
std::vector<Partition> hierarchy; 
コードは注意してご使用ください。

② オーディオスレッドでの処理 (processBlock)
オーディオスレッドでは、「小さなブロック」は即座に計算し、「大きなブロック」は計算予約（累積）だけを行います。
cpp
void processNonUniform(const double* input, double* output, int n) {
    // 1. 全階層のディレイライン（入力履歴）を更新
    updateAllDelayLines(input, n);

    // 2. Head部の計算 (即時実行)
    // ここが最小レイテンシを決定する
    applyFFTConvolution(hierarchy[0], currentInput, output);

    // 3. Body/Tail部の計算タイミング判定
    for (auto& part : hierarchy) {
        if (isTimeCompute(part)) {
            // [重要] 大きなFFTは MKL の非同期性を活かすか、
            // 別スレッドにタスクを投げる（JUCEのThreadPool等）
            asyncComputeFFT(part); 
        }
    }

    // 4. 前回の計算結果（バックグラウンドで完了したもの）を足し合わせる
    accumulatePriorResults(output, n);
}
コードは注意してご使用ください。

3. MKL 最適化のポイント（Non-uniform特有）
DFTI_NUMBER_OF_TRANSFORMS の活用:
Tail部などで複数のチャンネル（L/R）や複数のセグメントを一度に処理する場合、MKLの「バッチ処理」機能を使うと、1回ずつ呼ぶよりもキャッシュ効率が劇的に向上します。
vzMul による一括複素数乗算:
分割された各セグメントと入力の積を計算する際、MKL VMLの vzMul を使い、メモリ上の連続領域を一気に処理します。
マルチスレッド・スケジューリングの分離:
オーディオスレッド: Head部の計算のみ。
ワーカースレッド: Tail部の巨大なFFT計算。
ここで、Tail部の計算が終わるまで待ってはいけません。計算結果は「数ブロック後」に現在の出力に足し込まれる（Overlap-Addの拡張）設計にします。
4. 768kHzにおける注意点
768kHzでは、最小ブロックサイズが 256サンプルであっても、時間は 0.33ms しかありません。
Direct Formの併用: 最初の数サンプル〜64サンプル程度は、FFTを使わず AVX2 による Direct Convolution（時間領域での畳み込み）を行うことで、真のゼロレイテンシを実現できます。
