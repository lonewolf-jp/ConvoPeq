１. AVX2 / MKL 用の「非ゼロ係数」最適化テーブルへの変換
上記の係数には多くの 0.0 が含まれるため、AVX2のSIMD演算で無駄な 0.0 を計算しないように、「有効な係数のみ」を抽出した64byteアライメント済みのテーブルに作り替えます。

プリリンギングの監視:
-160dB仕様は遮断が非常に鋭いため、インパルス応答の「震え（リンギング）」が時間軸上で長く残ります。コンボルバーに送るデータの「立ち上がり」部分が削られないよう、フィルタ遅延分（N/2 サンプル）の先読みバッファを十分に確保してください。




struct OptimizedCoeffs {
    double* activeCoeffs; // 0.0を除いた係数
    int activeCount;

    OptimizedCoeffs(const std::vector<double>& raw) {
        std::vector<double> filtered;
        for (auto c : raw) {
            if (std::abs(c) > 1e-20) { // ほぼ0でないものを抽出
                filtered.push_back(c);
            }
        }
        activeCount = (int)filtered.size();
        // MKL/AVX2のために64byteアライメントで確保
        activeCoeffs = (double*)_aligned_malloc(activeCount * sizeof(double), 64);
        std::copy(filtered.begin(), filtered.end(), activeCoeffs);
    }
    
    ~OptimizedCoeffs() { _aligned_free(activeCoeffs); }
};
