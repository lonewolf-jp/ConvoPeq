１．768kHzという極限のレートで1023タップのフィルタを回す際、メモリアクセスの「不連続性」は致命的なオーバーヘッドとなります。SIMD（AVX2）の性能を100%引き出すための、「リニア・バッファ・スライディング」方式のディレイライン設計図を提示します。
通常のリングバッファ（剰余演算 % を使うもの）は、SIMDの連続ロードと相性が悪いため、プロ用DSPでは「パディング付きリニアバッファ」を採用します。
AVX2最適化ディレイライン（履歴バッファ）設計図
この設計では、バッファの末尾に「タップ数分のアライメント済みコピー領域（パディング）」を設けることで、条件分岐なしで連続した _mm256_loadu_pd を実行可能にします。

２．この設計図の「プロ用」としての価値
SIMDフレンドリー（連続性）:
_mm256_loadu_pd は、メモリが物理的に連続している時に最もスループットが出ます。リングバッファのような「末尾で先頭に戻る」処理をループ内から排除したことで、CPUのプリフェッチャ（先読み機能）が完璧に機能します。
MKL vdCopy の活用:
メモリ移動には C標準の memcpy ではなく、Intel CPUに最適化された MKL の vdCopy を使用します。これにより、OS内部でのデータ移動負荷を極限まで下げます。
768kHzの壁を突破:
1.3μsという制約下では、ポインタのインクリメントすら最小限にする必要があります。この「流し込み＋一括スライド」方式は、分岐予測ミスによるパイプラインストールを物理的に防ぎます。

/*==============================================================================
    OS_DelayLine_AVX2.h - v1.0
    AVX2の連続ロードを妨げない、超高速スライディングバッファ
==============================================================================*/

class alignas(64) OSDelayLine {
public:
    // 初期化: 最大ブロックサイズ + 最大タップ数 を確保
    void prepare(int maxBlockSize, int maxTaps) {
        historySize = maxTaps;
        // バッファ長 = ヒストリ分 + 処理ブロック分 + 境界保護用パディング
        const int totalSize = historySize + maxBlockSize + 16; 
        buffer = std::make_unique<AlignedBuffer<double>>(totalSize);
        std::fill_n(buffer->data, totalSize, 0.0);
        
        writePos = historySize; // 書き込み開始位置をヒストリの直後に設定
    }

    /**
     * AVX2カーネルへ、現在の処理ポインタを渡す
     * @param inputBlock   AudioThreadからの入力
     * @param numSamples   今回のブロックサイズ
     */
    void processBlock(const double* inputBlock, int numSamples, 
                      const OSStageCoeffs* stage, double* output) noexcept 
    {
        // 1. 新しい入力をヒストリの直後にコピー
        // MKLの vdCopy を使うと64byte alignの恩恵で最速
        vdCopy(numSamples, inputBlock, &buffer->data[writePos]);

        // 2. メイン演算ループ (AVX2カーネルの呼び出し)
        for (int i = 0; i < numSamples; ++i) {
            // 現在のサンプル位置 (buffer->data[writePos + i]) を「中心」として
            // 前後(Symmetric)のデータを AVX2 で一気にロードして演算
            output[i * 2] = buffer->data[writePos + i - stage->totalDelay]; // 偶数
            output[i * 2 + 1] = processSymmetricFIR_AVX2(
                &buffer->data[writePos + i], 
                stage->activeCoeffs, 
                stage->numActiveTaps, 
                0.5
            ); // 奇数
        }

        // 3. 次回のためにヒストリを更新 (バッファのスライド)
        // 処理が終わったブロックの末尾を、バッファの先頭（ヒストリ領域）へ移動
        vdCopy(historySize, &buffer->data[writePos + numSamples - historySize], buffer->data);
    }

private:
    std::unique_ptr<AlignedBuffer<double>> buffer;
    int writePos = 0;
    int historySize = 0;
};
