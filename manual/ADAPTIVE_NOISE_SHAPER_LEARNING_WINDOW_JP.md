# Adaptive Noise Shaper Learning ウィンドウ マニュアル

Adaptive Noise Shaper Learning ウィンドウは、**Adaptive 9th‑order** ノイズシェーパーの係数を、実際の再生信号に合わせて自動最適化（学習）するための専用画面です。

> **開き方**：Audio Settings 画面で Noise Shaper タイプを「Adaptive 9th‑order」に切り替え、「Adaptive learning…」ボタンをクリックします。

---

## 1. 学習モード（Learning Mode）

学習の「速さ」と「徹底度」を選択します。

| モード | 説明 |
|--------|------|
| **Shortest** | 最短（約10秒）。素早く大まかな係数を得たい場合に。 |
| **Short** | 短い（約30秒）。標準的な開始点として推奨。 |
| **Middle** | 中程度（約60秒）。より安定した結果が期待できます。 |
| **Long** | 長い（約120秒）。高精度な調整を行います。 |
| **Ultra** | 非常に長い（約300秒）。最高品質を求める場合に。 |
| **Continuous** | 継続的。Stop Learningボタンを押すか、アプリを終了するまで学習を続け、常に改善を試みます。 |

---

## 2. 学習コントロールボタン

| ボタン | 動作 |
|--------|------|
| **Start learning** | 選択中のモードで学習を開始します。過去の学習結果は破棄され、新たに最適化を始めます。 |
| **Stop learning** | 学習を中断します。現在の最良係数は維持され、次回「Resume learning」で再開できます。 |
| **Resume learning** | 以前中断した学習を再開します。過去の最良係数やCMA-ESの状態を引き継ぎます。 |

---

## 3. ステータス表示エリア

| 項目 | 説明 |
|------|------|
| **Status** | 現在の学習状態（Idle / Waiting for audio / Running / Completed / Error）を表示します。 |
| **Format** | 現在の処理サンプルレート（Hz）とディザビット深度を表示します。 |
| **Elapsed audio** | 学習に使用された累積再生時間（秒）。長いほど多くのデータで評価されています。 |
| **Phase** | 学習の進行フェーズ（1 / 2 / 3）。モードと経過時間に応じて自動的に変化し、探索の強度が変わります。 |
| **Generation** | CMA-ES の世代数（連続モード以外は総世代数も表示）。 |
| **Process count** | これまでに評価された候補係数の総数。 |
| **Training segments** | 現在の学習で使用されているオーディオセグメント数（多いほど多様な信号で評価）。 |
| **Best score** | これまでに見つかった最良の評価スコア（小さいほど良い）。 |
| **Latest score** | 最新の候補係数のスコア。 |

---

## 4. スコア推移グラフ（recent score）

- 横軸：学習ステップ（世代または評価回数）
- 縦軸：**対数スケール**のベストスコア（小さいほど良好）
- グラフには過去のベストスコア履歴が表示され、学習の収束状況を視覚的に確認できます。

---

## 5. 詳細設定（学習パラメータ）

| 項目 | 説明 |
|------|------|
| **CMA-ES Restarts** | 学習開始時に異なる乱数シードで複数回の初期探索を行う回数（1～10）。値を大きくすると、より良い初期値が見つかる可能性が高まりますが、学習開始までに時間がかかります。 |
| **Coeff Safety Margin** | 学習された反射係数（parcor）の絶対値の上限（0.3～0.95）。デフォルト 0.85。この値を小さくするとノイズシェイパーの安定性が向上しますが、音質改善効果が弱まります。 |
| **Enable Stability Check** | 有効にすると、不安定な係数（反射係数の絶対値が 1 に近いもの）に大きなペナルティを与え、学習から除外します。常に有効を推奨します。 |

---

## 6. 学習の流れ

1. **Waiting for audio**  
   ステレオのオーディオ再生が必要です。十分な長さの多様な信号（音楽など）を再生すると、自動的にトレーニングセグメントが収集されます。

2. **Running**  
   CMA-ES アルゴリズムが候補係数を生成し、実際の信号で評価しながら最適化を進めます。画面上でスコアとグラフが更新されます。

3. **Completed / Stopped**  
   これ以上最適化が望めない場合に学習が自動終了するか手動で停止すると、最良の係数が自動的に **Adaptive 9th‑order** ノイズシェーパーに適用され、以降のオーディオ処理で使用されます。学習結果は各サンプルレート・ビット深度・モードごとに保存され、次回起動時にも引き継がれます。

---

## 7. Adaptive 9th-order 学習の仕組み

Adaptive 9th-order ノイズシェーパーは **格子型（Lattice）構造** を持ち、**反射係数（PARCOR）** と呼ばれる9個のパラメータで動作します。学習機能は以下の手順で最適な反射係数を自動探索します。

1. **信号の収集**  
   ステレオ再生信号から、異なる音量レベル（-40dBFS ～ -10dBFS）と特性（トランジェント、トーナル、ブロードバンド）を持つ複数のオーディオセグメントを抽出します。

2. **CMA-ES 最適化**  
   Covariance Matrix Adaptation Evolution Strategy という進化的アルゴリズムを用いて、反射係数の候補を生成・評価・更新します。各世代で18個の候補を試し、その中から最も評価の高い6個を親として次の世代を生成します。

3. **評価（スコアリング）**  
   各候補係数を実際のノイズシェーパーに適用し、量子化誤差を分析します。評価には以下の要素が考慮されます：
   - 聴覚マスキング曲線に基づく周波数領域の重み付け
   - 時間領域の RMS 誤差
   - スペクトル平坦度（フラットネス）のペナルティ
   - 高域過剰エネルギーのペナルティ
   - トーナル成分（ピーク）のペナルティ

4. **学習フェーズ**  
   経過時間に応じて「探索（広範囲）→ 収束（狭範囲）→ 微調整」の3フェーズに切り替わり、効率的に最適解へと導きます。

5. **結果の保存と適用**  
   最良の反射係数はノイズシェーパーに即座に反映され、同時にディスクに保存されます。次回同じサンプルレート・ビット深度・学習モードで起動した際に自動的に読み込まれます。

この学習により、固定係数のノイズシェーパーでは得られない、入力信号に適応した高品位なノイズ整形が可能になります。

---

## 8. 注意事項

- 学習中は他のノイズシェーパータイプに切り替えると学習が中断されます（再開可能）。
- 学習には**ステレオ**の再生信号が必要です。無音や単純なテストトーンでは効果的な学習ができません。
- 学習中も通常のオーディオ処理は継続されますが、CPU 負荷がやや増加します。
- 学習結果は自動的に `%APPDATA%\ConvoPeq\learned_state.xml` に保存されます。手動でバックアップすることも可能です。
