## 設計原則

### 共通キー
全ログに callbackIndex, gen, seq を付与（v4から継続）。

### budget + expectedUs
全時間計測に budget% と expectedIntervalUs の両方を付与。
異なるバッファサイズ間でも比較可能に。

### 共有データの整数化
リングバッファの budget は double ではなく uint16_t permille で保持。
RT共有データは可能な限り整数型を使用。

### consumeAtomic ではなく load(relaxed)
診断専用値は consumeAtomic ではなく load(relaxed) で読み取る。
consumeAtomic のデフォルトは memory_order_acquire であり、診断用途では不要な同期コスト。

## G: CPU migration 記録（v5改善版）

### v4からの改善点
- audioCallbackEpochCounter の読取を consumeAtomic から load(relaxed) に変更
- （consumeAtomic は単なる load(acquire) だが、診断用途では relaxed で十分）

### コード
```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
{
    const uint32_t cpu = static_cast<uint32_t>(::GetCurrentProcessorNumber());
    const uint32_t prev = rtLocalState_.lastCallbackProcessor.load(
        std::memory_order_relaxed);
    if (prev != cpu) {
        rtLocalState_.lastCallbackProcessor.store(cpu, std::memory_order_relaxed);
        if (prev != UINT32_MAX) {
            convo::fetchAddAtomic(rtLocalState_.cpuMigrationCount,
                uint64_t{1}, std::memory_order_relaxed);
            // ★ consumeAtomic → load(relaxed) に変更
            const uint64_t cbIdx = rtLocalState_.audioCallbackEpochCounter.load(
                std::memory_order_relaxed);
            const uint64_t pubSeq = (runtimeWorld != nullptr)
                ? runtimeWorld->metadata.publicationSequence : 0;
            const uint64_t gen = (runtimeWorld != nullptr)
                ? static_cast<uint64_t>(runtimeWorld->generation) : 0;
            juce::Logger::writeToLog(
                "[CPU_MIG] callback=" + juce::String((int64_t)cbIdx)
                + " seq=" + juce::String((int64_t)pubSeq)
                + " gen=" + juce::String((int64_t)gen)
                + " cpu=" + juce::String((int)cpu)
                + " prev=" + juce::String((int)prev));
        }
    }
}
#endif
```

### 補足
audioCallbackEpochCounter は fetchAddAtomic でインクリメントされる atomic<uint64_t>。
consumeAtomic は単なる load(acquire) であり値を消費しないが、診断用途では relaxed で十分なため変更。

## B: Callback処理時間 リングバッファ（commit seq + uint16_t permille）

### v4からの改善点
- double budgetPct → uint16_t budgetPermille に整数化
- expectedIntervalUs も保持（ログ出力用）

### 構造体定義（AudioEngine.h / RTLocalState）
```cpp
static constexpr size_t kCallbackTimingSlots = 16;
struct CallbackTimingEntry {
    uint64_t callbackIndex = 0;
    uint64_t processTimeUs = 0;
    int64_t driftUs = 0;
    uint32_t cpu = UINT32_MAX;
    uint16_t budgetPermille = 0;     // ★ double→uint16_t（例: 18.4% → 184）
    uint16_t expectedIntervalUsLow = 0; // expected下位16bit（概算用）
    uint16_t expectedIntervalUsHigh = 0; // expected上位16bit
    std::atomic<uint64_t> sequence{0};   // commit seq
};
CallbackTimingEntry callbackTimingHistory[kCallbackTimingSlots];
std::atomic<uint64_t> callbackTimingWriteCount{0};
```

### 書き込み側
```cpp
const uint64_t wc = convo::fetchAddAtomic(
    rtLocalState_.callbackTimingWriteCount, uint64_t{1}, std::memory_order_relaxed);
const size_t idx = static_cast<size_t>((wc - 1) % kCallbackTimingSlots);
auto& entry = rtLocalState_.callbackTimingHistory[idx];
// 1) 非atomicフィールドを先に書く
entry.callbackIndex = thisCallbackIndex;
entry.processTimeUs = processTime;
entry.driftUs = rtLocalState_.lastCallbackDriftUs.load(std::memory_order_relaxed);
entry.cpu = rtLocalState_.lastCallbackProcessor.load(std::memory_order_relaxed);
// budget を permille（uint16_t）に変換
const uint16_t permille = static_cast<uint16_t>(
    std::min(999.0, std::round(budgetPct * 10.0)));
entry.budgetPermille = permille;
const uint64_t expected = static_cast<uint64_t>(expectedIntervalUs);
entry.expectedIntervalUsLow = static_cast<uint16_t>(expected & 0xFFFFu);
entry.expectedIntervalUsHigh = static_cast<uint16_t>((expected >> 16) & 0xFFFFu);
// 2) 最後に sequence を更新（commit）
entry.sequence.store(wc, std::memory_order_release);
```

### 読み取り側（XRUN時）
```cpp
if (xrunDetected) {
    const uint64_t wc = convo::consumeAtomic(
        rtLocalState_.callbackTimingWriteCount, std::memory_order_relaxed);
    for (uint64_t i = wc; i > wc - kCallbackTimingSlots && i > 0; --i) {
        const size_t idx = (i - 1) % kCallbackTimingSlots;
        const auto& e = rtLocalState_.callbackTimingHistory[idx];
        const uint64_t seq = e.sequence.load(std::memory_order_acquire);
        if (seq == 0) continue;
        const uint64_t cbIdx = e.callbackIndex;
        const uint64_t proc = e.processTimeUs;
        const int64_t drift = e.driftUs;
        const uint32_t cpu = e.cpu;
        const double pct = static_cast<double>(e.budgetPermille) / 10.0;
        const uint64_t expected = static_cast<uint64_t>(e.expectedIntervalUsLow)
            | (static_cast<uint64_t>(e.expectedIntervalUsHigh) << 16);
        if (e.sequence.load(std::memory_order_acquire) == seq) {
            juce::Logger::writeToLog(
                "[CB_HIST] idx=" + juce::String((int64_t)cbIdx)
                + " proc=" + juce::String((int64_t)proc)
                + " drift=" + juce::String((int64_t)drift)
                + " cpu=" + juce::String((int)cpu)
                + " budget=" + juce::String(pct, 1) + "%"
                + " expected=" + juce::String((int64_t)expected));
        }
    }
}
```

### 補足
expectedIntervalUs は 32bit に収まる（最大 ~42ms@512samples）が、将来の拡張性のため
uint64_t を 2x uint16_t に分割して保持。必要に応じて uint32_t 1つに統合可能。

## 仮説判定マトリクス（v5版）

| 観測パターン | 示唆する原因 |
| --- | --- |
| DriftUs大 + CPU_MIG有 | OSスケジューラ問題（CPU migration + 遅延） |
| DriftUs大 + CPU_MIG無 | OS/Driver DPC/ISR問題 |
| DriftUs小 + CB_SEQ変化無 + CONV_TIME大 + STCONV_TIME大 | FFT畳み込み遅延 |
| DriftUs小 + CB_SEQ変化無 + CONV_TIME大 + STCONV_TIME小 | Convolver管理処理遅延（RCU等） |
| DriftUs小 + EQ_TIME大 | EQ IIRフィルタ遅延 |
| DriftUs小 + CB_HIST漸増傾向 + budget増加 | メモリプレッシャーまたは経年劣化 |
| DriftUs小 + CB_HIST急増 + CPU_MIG有 | OS要因（コンテキストスイッチ） |
| 全計測値小 + budget余裕あり | 偶発的OSジッター |
