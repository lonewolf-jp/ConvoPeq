## 共通設計: タイムスタンプ接頭辞

すべての診断ログ行に `[HH:mm:ss.zzz]` 形式のタイムスタンプを付与する。これにより、異なるログ種別（VERIFY/MEM/XRUN/BUILD_PHASE 等）間の因果関係を時刻ベースで追跡できる。

### diagLog の拡張（推奨）

```cpp
namespace {
// 全ログに共通で付与する診断用接頭辞を生成する。
// - [HH:mm:ss.zzz] タイムスタンプ（壁時計、NTP補正の影響を受ける可能性あり）
// - Gen=N: 現在の RuntimeWorld の generation
// - Ticks=N: HighResolutionTicks（単調増加、イベント間隔の正確な計算に使用）
juce::String diagPrefix(uint64_t currentGeneration)
{
    const auto now = juce::Time::getCurrentTime();
    const auto timestamp = now.formatted("%H:%M:%S.")
        + juce::String(now.getMilliseconds()).paddedLeft('0', 3);
    const auto ticks = juce::Time::getHighResolutionTicks();
    return "[" + timestamp + "]"
        + " Gen=" + juce::String(static_cast<juce::int64>(currentGeneration))
        + " Ticks=" + juce::String(static_cast<juce::int64>(ticks));
}

void diagLog(const juce::String& message)
{
    DBG(message);
    juce::Logger::writeToLog(message);
}
} // namespace
```

**使用例**:

```cpp
// timerCallback() 内
const uint64_t gen = (runtimeWorld != nullptr)
    ? static_cast<uint64_t>(runtimeWorld->generation)
    : 0;
diagLog(diagPrefix(gen) + " [MEM] Private=...");
```

### 出力例

```
[12:18:31.224] Gen=8 Ticks=87456322315 [MEM] Private=2800MB WS=2200MB ...
[12:18:31.287] Gen=8 Ticks=87456322560 [BUILD_PHASE] build=95.9ms ...
[12:18:31.305] Gen=8 Ticks=87456323200 [XRUN#27] Callback=8.61ms ...
```

Gen は各ログ出力時点の RuntimeWorld generation、Ticks は HighResolutionTicks の生値。Gen で publish 世代との相関、Ticks で正確なイベント間隔を計算できる。

### RT安全性の注意

- `juce::Time::getCurrentTime()` および `juce::Time::getHighResolutionTicks()` は Timer/Worker Thread からのみ呼ぶ（non-RT safe）
- Audio Thread では一切呼ばない（XRUN検出部の `timestampTicks` は `juce::Time::getHighResolutionTicks()` を使用するが、これは Timer Thread での時刻再構成用であり、Audio Thread 内で壁時計を取得しているわけではない）
- `diagLog` は Timer/Worker Thread からのみ呼ばれる関数であるため安全

## 変更1: Retire/Backpressure ログ追加

### 現在の状態

`getRuntimeBackpressureTelemetry()` は定義されているが、**どこからも呼ばれていない**。
`retireQueueDepth_` / `fallbackQueueDepth_` / `quarantineResident_` 等の値は内部的に更新されているが、**ログに出力されていない**。

### 変更内容

#### 1-B. timerCallback に診断取得とログ出力を追加

**ファイル**: `src/audioengine/AudioEngine.Timer.cpp` — `timerCallback()`

`getRuntimeBackpressureTelemetry()` を呼び出し、値を前回と比較して変化があれば `[VERIFY] tx counters` に追記する。

```cpp
// timerCallback() 内の既存 VERIFY ブロックに追加（L107〜L176 付近）

// ★ 追加コード:
const auto backpressure = getRuntimeBackpressureTelemetry();
const bool backpressureChanged =
    backpressure.retireQueueDepth != rtAuxMutable_.debugLastReportedRetireQueueDepth
    || backpressure.fallbackQueueDepth != rtAuxMutable_.debugLastReportedFallbackQueueDepth
    || backpressure.quarantineResident != rtAuxMutable_.debugLastReportedQuarantineResident;

// 既存の lifecylce/rebuild 変化 OR backpressure 変化でログ出力
if (/* 既存条件 */ || backpressureChanged)
{
    // 既存の値更新に追加:
    rtAuxMutable_.debugLastReportedRetireQueueDepth = backpressure.retireQueueDepth;
    rtAuxMutable_.debugLastReportedFallbackQueueDepth = backpressure.fallbackQueueDepth;
    rtAuxMutable_.debugLastReportedQuarantineResident = backpressure.quarantineResident;

    // 既存の diagLog に追記:
    // 変更前:
    //   diagLog("[VERIFY] tx counters lifecycle(pub/ret/reclaim)=...");
    // 変更後（Gen/Ticks は diagPrefix で付与）:
    const uint64_t verifyGen = (runtimeWorld != nullptr)
        ? static_cast<uint64_t>(runtimeWorld->generation) : 0;
    diagLog(diagPrefix(verifyGen) + " [VERIFY] tx counters ..."
        + ... // 既存部分はそのまま
        + " retireDepth=" + juce::String(static_cast<juce::int64>(backpressure.retireQueueDepth))
        + " fallback=" + juce::String(static_cast<juce::int64>(backpressure.fallbackQueueDepth))
        + " quarantine=" + juce::String(static_cast<juce::int64>(backpressure.quarantineResident))
        + " maxRetire=" + juce::String(static_cast<juce::int64>(
            convo::consumeAtomic(maxPendingRetireObserved_, std::memory_order_acquire)))
        + " rebuildBacklog=" + juce::String(static_cast<juce::int64>(backpressure.rebuildBacklog)));
```

#### 1-C. 別枠: retirePressureLevel の単独ログ

変化が頻繁なため、`[VERIFY] tx counters` とは独立して `[BACKPRESSURE]` タグで出力してもよい。

```cpp
// ★ retirePressureLevel 変化時のみ出力（既存の AdaptiveCoeff / ADAPTIVE_SWITCH と同パターン）
const int pressureLevel = backpressure.retirePressureLevel;
if (pressureLevel != rtAuxMutable_.debugLastReportedRetirePressureLevel)
{
    rtAuxMutable_.debugLastReportedRetirePressureLevel = pressureLevel;
    diagLog(diagPrefix(verifyGen) + " [BACKPRESSURE] retirePressureLevel=" + juce::String(pressureLevel)
        + " saturationEnter=" + juce::String(static_cast<juce::int64>(backpressure.saturationEnterCount))
        + " saturationExit=" + juce::String(static_cast<juce::int64>(backpressure.saturationExitCount))
        + " maxDeferralEpochs=" + juce::String(static_cast<juce::int64>(backpressure.maxRetireDeferralEpochs))
        + " maxWallClockMs=" + juce::String(backpressure.maxRetireWallClockMs, 1));
}
```

### 変更の根拠

- `retireQueueDepth` と `PrivateBytes` の相関が確認できれば、メモリ3GBの原因がRuntime滞留か否かを特定できる
- `maxPendingRetireObserved_` は最大滞留数を記録しているが未出力
- `retirePressureLevel` は Admission が圧力を検出したかどうかを示す重要な指標

これにより「DSPCore準備からpublishキュー投入まで」の全体時間が可視化される。後続の `trySubmit` ログ（`publish SUCCEEDED`）と組み合わせれば、commit待ち時間も算出可能。

```cpp

// 1. prepare 先頭でタイマー開始
const double prepareStartMs = juce::Time::getMillisecondCounterHiRes();

// 2. 各ステップの開始前に前ステップからの経過時間を測定
//    例: ramp.prepare
//    変更前:
//    juce::Logger::writeToLog("[DSPCORE_PREPARE] calling ramp.prepare");

//    juce::Logger::writeToLog("[DSPCORE_PREPARE] ramp.prepare done");

static const auto elapsedSince = [](double fromMs) -> double {
    return juce::Time::getMillisecondCounterHiRes() - fromMs;

juce::Logger::writeToLog("[DSPCORE_PREPARE] calling ramp.prepare");
ramp.prepare(newSampleRate);
juce::Logger::writeToLog("[DSPCORE_PREPARE] ramp.prepare done: "
    + juce::String(elapsedSince(t0), 2) + "ms");
```

## 変更3: Windows API メモリ使用量の定期ログ

### 現在の状態

Windows `GetProcessMemoryInfo()` は使用されていない。Private Bytes / Working Set / PageFault が一切出力されていない。

### 変更内容

#### 3-A. 新規関数の追加

**ファイル**: `src/audioengine/AudioEngine.Timer.cpp`

ヘッダインクルード:
```cpp
#if JUCE_WINDOWS
#include <windows.h>
#include <psapi.h>
#pragma comment(lib, "psapi.lib")
#endif
```

メモリ診断関数:
```cpp
namespace {

#if JUCE_WINDOWS
struct ProcessMemoryInfo {
    uint64_t privateUsageMB = 0;
    uint64_t workingSetMB = 0;
    uint64_t pageFaultCount = 0;
};

ProcessMemoryInfo getProcessMemoryInfo() noexcept
{
    ProcessMemoryInfo info{};
    PROCESS_MEMORY_COUNTERS_EX pmc;
    pmc.cb = sizeof(pmc);
    if (GetProcessMemoryInfo(GetCurrentProcess(),
            reinterpret_cast<PROCESS_MEMORY_COUNTERS*>(&pmc), sizeof(pmc)))
    {
        info.privateUsageMB = static_cast<uint64_t>(pmc.PrivateUsage / (1024ULL * 1024ULL));
        info.workingSetMB = static_cast<uint64_t>(pmc.WorkingSetSize / (1024ULL * 1024ULL));
        info.pageFaultCount = static_cast<uint64_t>(pmc.PageFaultCount);
    }
    return info;
}
#else
// 非Windows: ダミー実装（常に0を返す）
ProcessMemoryInfo getProcessMemoryInfo() noexcept { return {}; }
#endif

} // anonymous namespace
```

#### 3-B. timerCallback への組み込み（1秒周期・常時出力）

**ファイル**: `src/audioengine/AudioEngine.Timer.cpp` — `timerCallback()`

1秒ごとに必ずメモリ情報を出力する。変化がない場合も出力することで、「メモリが安定していた時期」と「変化した時期」の両方を記録する。

```cpp
// timerCallback() 内、既存の [VERIFY] ブロックの後に追加
// 1秒ごとに必ずメモリ情報を出力（変化なしでも出力）

{
    static uint64_t lastMemLogTicks = 0;
    const auto nowTicks = juce::Time::getHighResolutionTicks();
    const auto ticksPerSec = juce::Time::getHighResolutionTicksPerSecond();
    if (nowTicks - lastMemLogTicks >= ticksPerSec)
    {
        lastMemLogTicks = nowTicks;
        const auto memInfo = getProcessMemoryInfo();
        const uint64_t pfDelta = (s_pageFaultBase > 0)
            ? (memInfo.pageFaultCount - s_pageFaultBase)
            : 0;
        s_pageFaultBase = memInfo.pageFaultCount;

        rtAuxMutable_.debugLastReportedPrivateUsageMB = memInfo.privateUsageMB;
        rtAuxMutable_.debugLastReportedWorkingSetMB = memInfo.workingSetMB;

        // ★ XRUN消費ループ用のメモリキャッシュも同時更新（4-Dで使用）
        s_cachedMemInfo = memInfo;

        diagLog(diagPrefix(verifyGen) + " [MEM] Private=" + juce::String(static_cast<juce::int64>(memInfo.privateUsageMB)) + "MB"
            + " WS=" + juce::String(static_cast<juce::int64>(memInfo.workingSetMB)) + "MB"
            + " PageFaults=" + juce::String(static_cast<juce::int64>(memInfo.pageFaultCount))
            + " PageFaultDelta=+" + juce::String(static_cast<juce::int64>(pfDelta)));
        }
    }
}
```

### 出力例

```
[MEM] Private=2800MB WS=2200MB PageFaults=4520 PageFaultDelta=+0
[MEM] Private=2850MB WS=2250MB PageFaults=4580 PageFaultDelta=+60   ← ページフォールト増加
[MEM] Private=3010MB WS=2350MB PageFaults=5120 PageFaultDelta=+540  ← ★ 急増
```

> **PageFaultDeltaの統一基準**: MEMログとXRUNログで `s_pageFaultBase` を共有する。初回のMEMログ出力時に基準値を設定し、以降のMEMログ・XRUNログは全てこの基準からの差分を `PageFaultDelta` として出力する。これにより両ログのPageFaultDeltaが同一基準で比較可能になる。

### 設計判断

- **1秒ごと・常時出力**: 100msごとではログが膨大になるため、変化を検出するのに十分な1秒間隔。変化なしでも毎秒出力することで、「メモリが安定していた時期」と「変化した時期」の両方を記録できる
- **Private Bytes優先**: Working SetよりPrivate Bytesの方がプロセス固有のメモリ使用量を正確に示す
- **PageFaultDeltaはMEMログ専用**: XRUNログには含めない。XRUN時点のPageFaultはTimer Threadでのスナップショットであり、発生時の正確な値ではないため混同を避ける

## 変更4: XRUN 検出（RT-safe 設計）

### 設計方針

**Audio Thread では一切の I/O・API 呼び出しを行わない。** XRUN イベントの検出と記録は以下の責務分離に従う。

| 処理 | スレッド | 内容 |
|------|---------|------|
| 検出 + push | **Audio Thread (RT)** | 閾値判定（constexpr 複数レベル）、LockFreeRingBufferへpush — ここで完結。Logger/API/メモリ確保一切なし |
| 読出 + 集約出力 | **Timer Thread (non-RT)** | リングバッファをpop、`GetProcessMemoryInfo`・`getRuntimeBackpressureTelemetry()` でXRUN時点の全関連情報を取得し、一行に集約して `diagLog` |

### 4-A. XRunEvent 構造体 + RTLocalState への追跡用変数追加

**ファイル**: `src/audioengine/AudioEngine.h` — `struct RTAuxMutable` の近く（XRunEvent）、`struct RTLocalState`（追跡用変数）

### 4-C. getNextAudioBlock での XRUN 検出（RT-safe）

**ファイル**: `src/audioengine/AudioEngine.Processing.AudioBlock.cpp` — `getNextAudioBlock()`

Audio Thread では閾値判定とリングバッファへの push のみを行う。絶対時刻・メモリ情報は付与しない。

> **RT安全性の保証**: この関数内で `Logger::writeToLog()` や Windows API を一切呼ばない。`juce::Time::getHighResolutionTicks()` は QueryPerformanceCounter と等価で十分軽量。すべての重い処理（Logger, GetProcessMemoryInfo, 文字列結合）は Timer Thread（4-D）に委譲する。

```cpp
// 関数先頭付近: メンバー変数で前回終了時刻を保持（static ではなく RTLocalState のメンバ）
// rtLocalState_.lastCallbackEndTicks  … 前回コールバック終了の HighResolutionTicks（atomic）
// rtLocalState_.xrunSequenceCounter   … XRUN連番カウンタ（atomic）

const auto t0 = juce::Time::getHighResolutionTicks();
const auto ticksPerSec = juce::Time::getHighResolutionTicksPerSecond();
const double expectedMs = (engineSampleRate > 0.0)
    ? static_cast<double>(numSamples) / engineSampleRate * 1000.0
    : 0.0;

// callback間隔（XRUNの主指標）
double intervalMs = 0.0;
if (convo::consumeAtomic(rtLocalState_.lastCallbackEndTicks, std::memory_order_relaxed) > 0)
{
    intervalMs = static_cast<double>(t0 - convo::consumeAtomic(rtLocalState_.lastCallbackEndTicks, std::memory_order_relaxed))
        * 1000.0 / static_cast<double>(ticksPerSec);
}
```

```cpp
// 関数末尾付近: callback終了時間を計測 + XRUN判定
const auto t1 = juce::Time::getHighResolutionTicks();
const double callbackMs = static_cast<double>(t1 - t0)
    * 1000.0 / static_cast<double>(ticksPerSec);

// ★ 複数レベルの閾値（constexpr、必要に応じて選択）
constexpr double kFixedMarginMs = 1.0;       // 最低保証マージン 1ms
constexpr double kRatioLoose = 1.5;          // 緩い閾値（旧設計から維持）
constexpr double kRatioModerate = 1.3;       // 中間閾値（推奨）
constexpr double kRatioStrict = 1.2;         // 厳しい閾値（初期診断向け）
// ★ 実際に使用する比率（運用中に変更可能にするため constexpr で一元管理）
constexpr double kRatioThreshold = kRatioStrict;

bool exceededCallback = false;
bool exceededInterval = false;

// 主判定: interval 超過（ドライバ遅延検出）
if (intervalMs > 0.0 && expectedMs > 0.0)
{
    // hybrid 閾値: max(expected * 1.2, 1ms)
    // ブロックサイズが小さいとき(64samples@192kHz=0.33ms)でも最低1msを確保し、
    // 大きいとき(4096samples=21ms)は比率で判定する
    const double intervalThreshold = std::max(expectedMs * kRatioThreshold, kFixedMarginMs);
    if (intervalMs > intervalThreshold)
        exceededInterval = true;
}

// 副判定: callback 時間超過（Audio Thread 内の処理遅延検出）
if (expectedMs > 0.0)
{
    // callback > max(expected * 1.2, 1ms)
    const double callbackThreshold = std::max(expectedMs * kRatioThreshold, kFixedMarginMs);
    if (callbackMs > callbackThreshold)
        exceededCallback = true;
}

if (exceededCallback || exceededInterval)
{
    XRunEvent ev;
    ev.timestampTicks = t1;
    ev.callbackMs = callbackMs;
    ev.intervalMs = intervalMs;
    ev.expectedMs = expectedMs;
    ev.generation = static_cast<int>(runtimeWorld->generation);
    // ★ XRUN発生時点のRetireDepthをatomic loadで取得（RT-safe、Timer側の100ms後ではない）
    ev.retireQueueDepth = convo::consumeAtomic(retireQueueDepth_, std::memory_order_relaxed);
    ev.sequenceNumber = convo::fetchAddAtomic(rtLocalState_.xrunSequenceCounter, uint64_t{1}, std::memory_order_acq_rel) + 1u;  // XRUN連番

    // RT-safe: LockFreeRingBuffer::push() — I/Oなし、API呼び出しなし
    // 満杯の場合は dropCount を増やして損失を追跡する
    if (!xRunBuffer.push(ev))
    {
        convo::fetchAddAtomic(xRunDropCount, static_cast<uint64_t>(1), std::memory_order_relaxed);
    }
}
convo::publishAtomic(rtLocalState_.lastCallbackEndTicks, t1, std::memory_order_release);
```

### 4-D. timerCallback での XRUN イベント読み出し（non-RT）

**ファイル**: `src/audioengine/AudioEngine.Timer.cpp` — `timerCallback()`

Timer スレッドがリングバッファを消費し、必要な情報（メモリ・Retire）を取得してログ出力する。

```cpp
// timerCallback() 内、既存のブロック後に追加
// XRUN リングバッファを消費（100ms毎）

{
    // ★ タイマー開始時の基準点（スリープ・休止対策）
    //    HighResolutionTicks と CurrentTime の対応を1度だけ保存し、
    //    そこからの差分でイベント時刻を算出する。
    //    これにより、PCスリープや高分解能タイマ基準変化の影響を軽減する。
    struct TimeReference {
        uint64_t baseTicks = 0;
        juce::Time baseTime;
        bool initialized = false;
    };
    static TimeReference timeRef;

    if (!timeRef.initialized)
    {
        timeRef.baseTicks = juce::Time::getHighResolutionTicks();
        timeRef.baseTime = juce::Time::getCurrentTime();
        timeRef.initialized = true;
    }

    // ★ PageFault の共通基準値（MEMログとXRUNログで共有）
    static uint64_t s_pageFaultBase = 0;

    // ★ ログ出力時点の generation（diagPrefix で使用）
    const uint64_t currentWorldGen = (runtimeWorld != nullptr)
        ? static_cast<uint64_t>(runtimeWorld->generation)
        : 0;

    // ★ 前回消費以降のXRUN drop count を読み取ってリセット
    const uint64_t xRunDropped = convo::exchangeAtomic(xRunDropCount, 0, std::memory_order_acq_rel);
    if (xRunDropped > 0)
    {
        diagLog(diagPrefix(currentWorldGen) + " [XRUN] dropped="
            + juce::String(static_cast<juce::int64>(xRunDropped))
            + " events (ring buffer full)");
    }

    // ★ メモリ情報のキャッシュ: MEMブロック（3-B）またはこのブロック冒頭で1度だけ取得。
    //    XRUNイベントごとに GetProcessMemoryInfo() を呼ぶと Timer スレッドとはいえ負荷になるため、
    //    最新のキャッシュを参照する。キャッシュが未初期化の場合はここで初回取得する。
    static ProcessMemoryInfo s_cachedMemInfo {};
    static bool s_memCacheInitialized = false;
    if (!s_memCacheInitialized)
    {
        s_cachedMemInfo = getProcessMemoryInfo();
        s_memCacheInitialized = true;
    }
    // ★ MEMブロック（3-B）の末尾で s_cachedMemInfo も更新される（一体化）

    XRunEvent ev;
    while (xRunBuffer.pop(ev))
    {
        // Timer Thread で安全に API + Logger を呼ぶ
        // ★ メモリ情報はキャッシュ済みの s_cachedMemInfo を使用（毎回のAPI呼び出し回避）
        const auto& memInfo = s_cachedMemInfo;
        const auto ticksPerSec = juce::Time::getHighResolutionTicksPerSecond();

        // ★ イベント発生時刻を XRunEvent の timestampTicks から再構成
        //    TimeReference は起動時に1度だけ取得した壁時計と HighResolutionTicks の対応。
        //    ここから経過時間を計算することで、XRUN発生時刻（Timer起動時ではない）を
        //    正確に復元する。
        //    注: 壁時計は NTP 同期や DST の影響を受ける可能性があるため、
        //    正確なイベント間隔の計算には HighResolutionTicks（単調増加）を使用すること。
        const double elapsedFromRefMs = (ev.timestampTicks > timeRef.baseTicks)
            ? static_cast<double>(ev.timestampTicks - timeRef.baseTicks) * 1000.0 / static_cast<double>(ticksPerSec)
            : 0.0;
        const juce::Time eventTime = timeRef.baseTime
            + juce::RelativeTime::milliseconds(elapsedFromRefMs);
        const double uptimeMs = (ev.timestampTicks > 0)
            ? static_cast<double>(ev.timestampTicks) * 1000.0 / static_cast<double>(ticksPerSec)
            : 0.0;

        // ★ PageFaultDelta: MEMログとXRUNログで共通の s_pageFaultBase を使用
        //    初回は基準設定のみ、2回目以降は差分を出力
        //    注意: PageFaultDelta は XRUN 時点の値ではなく、Timer Thread 処理時点の値。
        //    XRUNログからは分離し、MEMログ（1秒ごと常時出力）でのみ扱う。
        if (s_pageFaultBase == 0)
        {
            s_pageFaultBase = memInfo.pageFaultCount;
        }
        const uint64_t pfDelta = memInfo.pageFaultCount - s_pageFaultBase;
        s_pageFaultBase = memInfo.pageFaultCount;

        // ★ XRUN 検出直後に Timer Thread が取得したスナップショット情報
        //    （注意: Private/WS は XRUN 発生時の値ではなく、
        //     Timer Thread 処理時点のスナップショットである。
        //     100ms以内の誤差はあるが、RetireDepth/Generation は
        //     Audio Thread から受け取った正確な値。）
        const auto backpressure = getRuntimeBackpressureTelemetry();
        const auto lifecycle = getRuntimeLifecycleDiagnostics();  // publishTotal 用
        const auto* currentWorld = getRuntimeWorldFromReadHandle(runtimeReadHandle);
        const auto* activeDsp = resolveActiveRuntimeDSPFromRuntimeWorldOnly(runtimeReadHandle);
        const auto* fadingDsp = resolveFadingRuntimeDSPFromRuntimeWorldOnly(runtimeReadHandle);
        const int activeCount = (activeDsp != nullptr) ? 1 : 0;
        const int fadingCount = (fadingDsp != nullptr) ? 1 : 0;
        const uint64_t currentWorldRev = (currentWorld != nullptr)
            ? static_cast<uint64_t>(currentWorld->generation)
            : 0;

        // ★ XRUNレベル: 実際の Interval/Expected 比から判定
        //    Strict: ratio < 1.3, Moderate: ratio < 1.5, Loose: ratio >= 1.5
        juce::String levelStr = "Loose";
        if (ev.expectedMs > 0.0 && ev.intervalMs > 0.0)
        {
            const double actualRatio = ev.intervalMs / ev.expectedMs;
            if (actualRatio < 1.3)
                levelStr = "Strict";
            else if (actualRatio < 1.5)
                levelStr = "Moderate";
        }

        // ★ XRUN#N 形式の連番（XRUNイベントの一意識別子）
        const juce::String xrunId = "XRUN#" + juce::String(static_cast<juce::int64>(ev.sequenceNumber));

        diagLog(diagPrefix(currentWorldRev) + " [" + xrunId + "]"
            + " Callback=" + juce::String(ev.callbackMs, 2) + "ms"
            + " Interval=" + juce::String(ev.intervalMs, 2) + "ms"
            + " Expected=" + juce::String(ev.expectedMs, 2) + "ms"
            + " Level=" + levelStr
            + " EventGen=" + juce::String(ev.generation)
            + " RetireDepth=" + juce::String(static_cast<juce::int64>(ev.retireQueueDepth))
            + " Private=" + juce::String(static_cast<juce::int64>(memInfo.privateUsageMB)) + "MB"
            + " WS=" + juce::String(static_cast<juce::int64>(memInfo.workingSetMB)) + "MB"
            + " World=" + juce::String(activeCount) + "/" + juce::String(fadingCount)
            + "/" + juce::String(static_cast<juce::int64>(backpressure.retireQueueDepth))
            + " Pressure=" + juce::String(backpressure.retirePressureLevel)
            + " PublishTotal=" + juce::String(static_cast<juce::int64>(lifecycle.publishCount)));
    }
}
```

### 4-E. publish世代の追跡（getNextAudioBlock内）

**ファイル**: `src/audioengine/AudioEngine.Processing.AudioBlock.cpp`

### 設計判断

### 設計判断

| 判断 | 理由 |
|------|------|
| **LockFreeRingBuffer 使用** | NoiseShaperLearner で実績のある SPSC lock-free キュー。`push()` は atomic 操作のみで RT-safe |
| **容量 64** | Timer が 100ms 毎に消費するため、64個で十分な余裕。最悪ケース（1ms毎にXRUN=100個/100ms）でもカバー可能。満杯時の損失は `dropCount` で追跡 |
| **XRUN時刻は timestampTicks から再構成** | XRUN発生時の `ev.timestampTicks` と起動時の `TimeReference` から壁時計を復元。Timer 起動時刻ではなく XRUN 発生時刻を正確に記録 |
| **static変数不使用 → RTLocalState メンバー** | `lastCallbackEndTicks`, `xrunSequenceCounter`, `lastActivatedGeneration` を `RTLocalState` の `std::atomic` メンバーとして宣言。複数Engine・テスト・将来のプラグイン版に安全対応 |
| **XRUN drop count** | RingBuffer満杯時のイベント損失を `xRunDropCount` atomic で追跡。Timer Thread が消費時に読み取り・リセットし、`[XRUN] dropped=N` としてログ出力 |
| **Audio Thread は push のみ** | XRUN検出の責務は最小限に。`juce::Logger::writeToLog()` は Timer Thread でのみ呼ぶ。Audio Thread では決して I/O・Logger・API を呼ばない（Practical Stable ISR Bridge Runtime の原則） |
| **XRunEvent 最小化** | Audio Thread からは `ticks`, `callbackMs`, `intervalMs`, `expectedMs`, `generation`, `retireQueueDepth` のみ伝達。残りの情報は Timer Thread が `getRuntimeBackpressureTelemetry()` / `getRuntimeLifecycleDiagnostics()` 経由で取得して集約 |
| **Interval を主判定** | callback時間が正常でもドライバ遅延は interval 超過で検出可能 |
| **hybrid 閾値** | `max(expected * ratio, 1ms)` — ブロックサイズに依存せず、小ブロックでも最低1msのマージンを確保 |
| **複数閾値レベル** | `constexpr kRatioStrict=1.2, kRatioModerate=1.3, kRatioLoose=1.5` を用意。用途に応じて選択可能。ログには実際の比から判定した Level（Strict/Moderate/Loose）を出力 |
| **Generation は XRUN時点の直接値** | `runtimeWorld->generation` を XRUN 検出時に直接読み取る。static変数 `s_xrunGeneration` は不要。`retireDepth` も同様に `convo::consumeAtomic(retireQueueDepth_)` で XRUN 時点の正確な値を保持 |
| **XRUN集約スナップショット** | XRUN 1件につき Private/WS/World/Pressure/PublishTotal を同一行で出力（ただし PageFaultDelta は含めない — MEMログ（1秒ごと常時出力）で別途扱う） |
| **PublishTotal** | `getRuntimeLifecycleDiagnostics().publishCount` をXRUNログに含める。累積publish回数が分かれば、XRUN発生前後のpublish頻度を推定可能 |
| **Memory情報は「スナップショット」** | Private/WS/PageFault は `GetProcessMemoryInfo()` 呼び出し時点の値（XRUN検出から約100ms以内）。正確なXRUN発生時の値ではないことを明記 |
| **全ログにタイムスタンプ** | `[HH:mm:ss.zzz]` を全診断ログに付与。異なるログ種別間の因果関係を時刻ベースで追跡可能 |

統合するとログ行数が減り解析の一貫性が向上するが、変化条件が異なる（WORLDはRetireDepthの変化でも発火するため、VERIFY条件と完全一致しない可能性がある）。状況に応じて選択する。

### WORLDログの解釈（解析上の目安）

`RetireQueue` = retire queue の長さ。`Quarantine` = 問題が検出され隔離された Runtime 数。合計が実際の滞留数を近似する。

| World状態 | 解釈 |
|----------|------|
| Active=1, Fading=0, RetireQueue=0, Quarantine=0 | 安定状態（Runtime滞留なし） |
| Active=1, Fading=1, RetireQueue=0, Quarantine=0 | クロスフェード中（正常） |
| Active=1, Fading=0, RetireQueue=1-2, Quarantine=0 | 軽度滞留（許容範囲） |
| Active=1, Fading=0/1, RetireQueue≥3, Quarantine=0 | 滞留あり（上昇傾向がある場合、音飛び原因の可能性） |
| Active=1, Fading=0/1, RetireQueue≥1, Quarantine≥1 | 滞留＋隔離あり（Runtime解放が停滞している可能性大） |

> **注**: これらの数値は解析上の目安であり、固定的な閾値ではない。`RetireQueue` は retire queue の長さ、`Quarantine` は隔離された Runtime 数を示す。両者の合計が実際に滞留している Runtime のおおよその数を表す。実際の影響はIRサイズ・サンプルレート・publish速度により変動する。XRUNログとの突き合わせで総合判断すること。

## 変更6: publish-to-activate 遅延の可視化（ACTIVATE）

### 現在の状態

publish が成功しても、Audio Thread が実際に新しい RuntimeWorld を読み取り始めるまでには RCU の切り替え待ち時間が発生する。この遅延が大きい場合、音飛びの原因になりうる。

### 変更内容

**ファイル**: `src/audioengine/AudioEngine.Processing.AudioBlock.cpp` — `getNextAudioBlock()`

Audio Thread が新しい generation の RuntimeWorld を検出した時刻を記録する。

```cpp
// ★ runtimeWorld 取得直後（L136付近）に ACTIVATE 検出を追加

// Audio Thread で、参照している RuntimeWorld の generation が変わったことを検出
// （rtLocalState_.lastActivatedGeneration に前回値を保持し、変化時のみイベントを発行）
const uint64_t currentGeneration = (runtimeWorld != nullptr)
    ? static_cast<uint64_t>(runtimeWorld->generation)
    : 0;
const uint64_t prevActivated = convo::consumeAtomic(
    rtLocalState_.lastActivatedGeneration, std::memory_order_relaxed);

if (currentGeneration != prevActivated && currentGeneration > 0)
{
    convo::publishAtomic(rtLocalState_.lastActivatedGeneration,
        currentGeneration, std::memory_order_release);

    // ★ 変更4と同じ xRunBuffer を流用（XRunEvent を ACTIVATE 通知としても使用）
    //    タイムスタンプと generation のみを記録
    XRunEvent ev;
    ev.timestampTicks = juce::Time::getHighResolutionTicks();
    ev.generation = static_cast<int>(currentGeneration);

    // RT-safe push（リングバッファ満杯時は損失を許容）
    xRunBuffer.push(ev);
}
```

### Timer Thread 側の出力

**ファイル**: `src/audioengine/AudioEngine.Timer.cpp` — `timerCallback()`

XRUN 消費ループ内で ACTIVATE イベントを識別して出力する。XRUN と ACTIVATE は同じリングバッファを共有するため、区別方法が必要。

**識別方法**: `XRunEvent.callbackMs == 0.0 && XRunEvent.intervalMs == 0.0` を ACTIVATE とみなす（XRUN では有効な値が入るため）。

```cpp
// xRunBuffer.pop(ev) ループ内、XRUN出力の前に追加

// ★ ACTIVATE イベントの識別（callbackMs=0, intervalMs=0 が ACTIVATE）
if (ev.callbackMs == 0.0 && ev.intervalMs == 0.0)
{
    // 起動時（generation==0）からの経過時間を uptime として計算
    const double activateUptimeMs = (ev.timestampTicks > 0)
        ? static_cast<double>(ev.timestampTicks) * 1000.0
            / static_cast<double>(juce::Time::getHighResolutionTicksPerSecond())
        : 0.0;

    diagLog(diagPrefix(currentWorldRev) + " [ACTIVATE] Generation=" + juce::String(ev.generation)
        + " Uptime=" + juce::String(activateUptimeMs, 1) + "ms");
    continue;  // 以降の Private/WS/Pressure 取得はスキップ
}

// ★ 以降は既存の XRUN 出力処理
```

### 設計判断

| 判断 | 理由 |
|------|------|
| **XRUN と同じリングバッファを流用** | ACTIVATE は高々1秒に数回しか発生しない。XRUN が32の容量に対して余裕がある。別バッファを追加するよりシンプル |
| **識別方法** | `callbackMs==0 && intervalMs==0` で識別。XRUN では両方とも正の値が入るため衝突しない |
| **Audio Thread で直接 generation 比較** | 静的な前回値との比較のみで、重い処理は一切なし。`runtimeWorld` は既に RCU 読取済み |

### 出力例

```
[12:18:31.300] [ACTIVATE] Generation=18 Uptime=52100.3ms
```

これにより、`[BUILD_PHASE]` の e2e 完了時刻と `[ACTIVATE]` の時刻を比較することで、publish-to-activate 遅延が算出できる。

```
publish queued at 12:18:31.250 (BUILD_PHASE e2e)
activate at     12:18:31.300 (ACTIVATE)
→ delay = 50ms
```

## マクロ化: CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS

### 設計方針

すべての診断ログ（VERIFY, BACKPRESSURE, WORLD, MEM, XRUN, DSPCORE_PREPARE, BUILD_PHASE）は、**コンパイル時マクロで完全に無効化できる**ようにする。Releaseビルドではコード自体が生成されない。

### マクロ定義

**新規ファイル**: `src/DiagnosticsConfig.h`

```cpp
#pragma once

// Runtime診断ログの有効/無効を制御する。
// 1: 有効（全診断ログが出力される）
// 0: 無効（診断コードはコンパイルされない）
//
// CMakeLists.txt で -DCONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS=1 として定義することも可能。
#ifndef CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
#define CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS 0
#endif
```

**CMakeLists.txt** に追加（L566〜L576 の `target_compile_definitions(ConvoPeq PRIVATE ...)` ブロック内）:

```cmake
option(CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS "Enable runtime diagnostic logging" OFF)
if(CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS)
    target_compile_definitions(ConvoPeq PRIVATE CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS=1)
endif()
```

既存の `CONVOPEQ_ENABLE_CONVOLVER_SPLIT_*` 定義群と同ブロックに追加することで一貫性を保つ。

### 適用パターン

マクロ化には2つの方式を併用する。

**方式A: `#if` ブロック（複数行の診断コードを囲む場合）**

```cpp
// 既存コードと同パターンの #if / #endif で囲む
// 例: AudioEngine.Timer.cpp

#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
namespace {
void diagLog(const juce::String& message)
{
    // ★ 全診断ログに [HH:mm:ss.zzz] タイムスタンプ接頭辞を付与
    const auto now = juce::Time::getCurrentTime();
    const auto timestamp = now.formatted("%H:%M:%S.")
        + juce::String(now.getMilliseconds()).paddedLeft('0', 3);
    DBG("[" + timestamp + "] " + message);
    juce::Logger::writeToLog("[" + timestamp + "] " + message);
}
} // namespace
#endif

void AudioEngine::timerCallback()
{
    // ... 既存コード（常時実行） ...

#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
    // ★ VERIFY tx counters
    {
        // ... 本設計書の変更1に相当 ...
    }
    // ★ WORLD ログ
    {
        // ... 本設計書の変更5に相当 ...
    }
#endif
}
```

**方式B: `RUNTIME_DIAG_LOG(x)` マクロ（単一の diagLog 呼び出しを囲む場合）**

```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
#define RUNTIME_DIAG_LOG(x) diagLog(x)
#else
#define RUNTIME_DIAG_LOG(x) do {} while(false)
#endif

// 使用例:
RUNTIME_DIAG_LOG("[BACKPRESSURE] retirePressureLevel=" + juce::String(pressureLevel));
```

これにより、`#if` ブロックの入れ子を減らし、コードの可読性を維持できる。

### 注意点

- **`DSPCoreLifecycle.cpp` の29箇所の `Logger::writeToLog` 直接呼び出し**: `#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS` で囲み、かつ `diagLog()` 経由に書き換える（タイムスタンプ統一のため）
- **`Retire.cpp` / `Learning.cpp`**: 両ファイルに `diagLog` が定義されていないため、`#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS` ブロック内での `diagLog()` 追加定義が必要
- **`AudioEngine.h` のインライン関数内の直接呼び出し**: `diagLog()` がスコープにない場合は `Logger::writeToLog` のままでも可。ただし `#if` ブロックでの囲みは必須

### 影響

| 状態 | バイナリサイズ | 実行時オーバーヘッド |
|------|-------------|-------------------|
| `=0` (Release) | 診断コードはコンパイルされない | ゼロ |
| `=1` (Debug) | 診断コードが含まれる | Timer callback 内の文字列生成 + Logger I/O |

### 細分化（必要に応じて）

現在は単一マクロで十分だが、将来的に細分化が必要な場合は以下の構成が可能:

```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
// すべて有効
#else
// 個別に有効化
#endif

// 個別マクロ例（必要になるまで定義しない）:
// CONVOPEQ_ENABLE_XRUN_DIAGNOSTICS
// CONVOPEQ_ENABLE_MEMORY_DIAGNOSTICS
// CONVOPEQ_ENABLE_RUNTIME_VERIFY
```
