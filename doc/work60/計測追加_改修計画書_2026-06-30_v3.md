## A: コールバック受信タイムスタンプ（最重要）

### 目的
OS/Driver → MMCSS → JUCE callback dispatch 区間のジッターを可視化。

### 設計上の注意
- atomic<double> 禁止 → atomic<int64_t> で整数μs保持
- exchangeAtomic 不使用 → load + store
- **memory_order は relaxed で十分**（診断専用値、他スレッドとの同期対象外）

### コード
```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
{
    const uint64_t nowUs = convo::getCurrentTimeUs();
    const uint64_t prevEntryUs = rtLocalState_.lastCallbackEntryUs.load(
        std::memory_order_relaxed);
    rtLocalState_.lastCallbackEntryUs.store(
        nowUs, std::memory_order_relaxed);
    if (prevEntryUs > 0 && numSamples > 0) {
        const double sr = getRuntimeSampleRateHzFromWorld(
            runtimeReadHandleRef, 0.0);
        if (sr > 0.0) {
            const double expectedUs =
                static_cast<double>(numSamples) / sr * 1e6;
            const int64_t driftUs = static_cast<int64_t>(
                static_cast<double>(static_cast<int64_t>(nowUs - prevEntryUs))
                - expectedUs);
            rtLocalState_.lastCallbackDriftUs.store(
                driftUs, std::memory_order_relaxed);
        }
    }
}
#endif
```

### メンバ変数（AudioEngine.h / RTLocalState）
```cpp
std::atomic<uint64_t> lastCallbackEntryUs { 0 };
std::atomic<int64_t> lastCallbackDriftUs { 0 };
```

### ログ出力
既存 [XRUN] ブロック内で relaxed load して追記:
```cpp
const int64_t driftUs = rtLocalState_.lastCallbackDriftUs.load(
    std::memory_order_relaxed);
log += " DriftUs=" + juce::String(static_cast<juce::int64>(driftUs));
```

## G: CPU migration 記録

### 目的
GetCurrentProcessorNumber() でCPUコア遷移を検出。

### 設計上の注意
- **memory_order は relaxe**d で十分
- **exchange ではなく load + 比較 + store** で実装
- CPU が変わった時のみ [CPU_MIG] ログ出力（ログ量抑制）

### コード
```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
{
    const uint32_t cpu = static_cast<uint32_t>(::GetCurrentProcessorNumber());
    const uint32_t prev = rtLocalState_.lastCallbackProcessor.load(
        std::memory_order_relaxed);
    if (prev != cpu) {
        rtLocalState_.lastCallbackProcessor.store(
            cpu, std::memory_order_relaxed);
        if (prev != UINT32_MAX) {
            convo::fetchAddAtomic(rtLocalState_.cpuMigrationCount,
                uint64_t{1}, std::memory_order_relaxed);
            juce::Logger::writeToLog(
                "[CPU_MIG] CPU=" + juce::String(static_cast<int>(cpu))
                + " prev=" + juce::String(static_cast<int>(prev)));
        }
    }
}
#endif
```

## H: callback開始時 publicationSequence（新規追加）

### 目的
各callback開始時の `runtimeWorld->metadata.publicationSequence` を記録し、
Publish→Observe→Callback の伝搬経路を精密に追跡する。

既に `publicationSequence` は RuntimeWorld の metadata に存在するため
(AudioBlock.cpp:303 で Observe 検出に使用済み)、callback 冒頭でも読み取るのみ。

### コード
```cpp
// ★ H: callback開始時 publicationSequence 記録
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
{
    if (runtimeWorld != nullptr) {
        const uint64_t seq = runtimeWorld->metadata.publicationSequence;
        const uint64_t prevSeq = convo::exchangeAtomic(
            rtLocalState_.lastCallbackPublicationSeq,
            seq, std::memory_order_relaxed);
        if (seq != prevSeq && prevSeq > 0) {
            // publicationSequence 変化時のみログ（Observe検出と重複しない）
            // DSP_TIMING とは別に callback 側の視点を記録
        }
    }
}
#endif
```

### メンバ変数
```cpp
std::atomic<uint64_t> lastCallbackPublicationSeq { 0 };
```

### 期待効果
- 各callbackがどのpublicationSequenceを参照しているかが可視化
- Publish→Runtime→Observe→Activate の遅延を sequence 単位で追跡可能
- 既存の DSP_TIMING と組み合わせて完全な伝搬経路を構築

## C: ConvolverProcessor::process() 実行時間

### 目的
ConvolverProcessor全体の実時間測定。

### 閾値設計（重要）
**固定値禁止。割合＋絶対値のハイブリッド**:
```cpp
const double ratioThreshold = expectedIntervalUs * 0.2;
constexpr double kAbsMinThresholdUs = 300.0;  // 最低300us
const double thresholdUs = std::max(ratioThreshold, kAbsMinThresholdUs);
if (static_cast<double>(convElapsedUs) > thresholdUs) { ... }
```

### 根拠
- 64sample@96kHz: 予算667us、20%=133us → 絶対値300usでクリップ
- 4096sample: 予算42.7ms、20%=8.5ms → 割合が支配
- このハイブリッドにより全バッファサイズで適切な感度を維持

## D: EQ::process() 実行時間（+ band数）

### 目的
EQ IIRフィルタ処理時間測定。**band数を併記して処理規模との相関を解析**。

### コード（DSPCoreDouble.cpp / DSPCoreFloat.cpp）
```cpp
#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
    const uint64_t eqStartUs = convo::getCurrentTimeUs();
#endif

    // 既存のEQ処理...

#if CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS
    const uint64_t eqElapsedUs = convo::getCurrentTimeUs() - eqStartUs;
    // ハイブリッド閾値: max(expectedIntervalUs * 0.1, 100.0)
    if (static_cast<double>(eqElapsedUs) > thresholdUs)
    {
        // 有効バンド数をカウント（eqParamsToUse から）
        int activeBands = 0;
        if (eqParamsToUse != nullptr) {
            for (int i = 0; i < 20; ++i) {
                if (eqParamsToUse->bands[i].enabled
                    && std::abs(eqParamsToUse->bands[i].gain) > 0.01f)
                    ++activeBands;
            }
        }
        juce::String log("[EQ_TIME] processUs=");
        log += juce::String(static_cast<juce::int64>(eqElapsedUs));
        log += " bands=" + juce::String(activeBands);
        diagLog(log);
    }
#endif
```

## B: Callback処理時間 リングバッファ保持

### 目的
XRUN発生時のみの計測ではなく、**直前16callback分の処理時間をリングバッファで保持**し、
XRUN発生時にまとめて出力する。

### 設計方針
既存の `PublishTimingEntry` ring buffer（kPublishTimingSlots=16）と同パターン。

### メンバ変数（RTLocalState）
```cpp
static constexpr size_t kCallbackTimingSlots = 16;
struct CallbackTimingEntry {
    uint64_t callbackIndex = 0;
    uint64_t processTimeUs = 0;
    uint64_t driftUs = 0;
    uint32_t cpu = UINT32_MAX;
};
CallbackTimingEntry callbackTimingHistory[kCallbackTimingSlots];
std::atomic<uint64_t> callbackTimingWriteCount { 0 };
```

### 記録（getNextAudioBlock 末尾付近）
```cpp
const uint64_t wc = convo::fetchAddAtomic(
    rtLocalState_.callbackTimingWriteCount,
    uint64_t{1}, std::memory_order_relaxed);
const size_t idx = static_cast<size_t>((wc - 1) % kCallbackTimingSlots);
auto& entry = rtLocalState_.callbackTimingHistory[idx];
entry.callbackIndex = thisCallbackIndex;
entry.processTimeUs = processTime;
entry.driftUs = static_cast<uint64_t>(std::abs(
    rtLocalState_.lastCallbackDriftUs.load(std::memory_order_relaxed)));
entry.cpu = rtLocalState_.lastCallbackProcessor.load(std::memory_order_relaxed);
```

### XRUN発生時の出力
既存の [XRUN] ブロック内で直前数エントリをまとめて出力:
```cpp
if (xrunDetected) {
    const uint64_t wc = convo::consumeAtomic(
        rtLocalState_.callbackTimingWriteCount, std::memory_order_relaxed);
    const uint64_t start = (wc > kCallbackTimingSlots)
        ? wc - kCallbackTimingSlots : 0;
    for (uint64_t i = start; i < wc; ++i) {
        const auto& e = rtLocalState_.callbackTimingHistory[
            i % kCallbackTimingSlots];
        if (e.callbackIndex > 0)
            juce::Logger::writeToLog(
                "[CB_HIST] idx=" + juce::String((int64_t)e.callbackIndex)
                + " proc=" + juce::String((int64_t)e.processTimeUs)
                + " drift=" + juce::String((int64_t)e.driftUs)
                + " cpu=" + juce::String((int)e.cpu));
    }
}
```

## 仮説判定マトリクス（拡張版）

| 観測パターン | 原因 |
| --- | --- |
| DriftUs大 + CPU_MIG有 | OSスケジューラ問題（CPU migration + 遅延） |
| DriftUs大 + CPU_MIG無 | OS/Driver DPC/ISR問題 |
| DriftUs小 + PubSeq変化無 + CONV_TIME大 | Convolver処理遅延（要因はC2で分離） |
| DriftUs小 + EQ_TIME大 | EQ IIRフィルタ遅延 |
| DriftUs小 + CB_HIST漸増傾向 | メモリプレッシャーまたは経年劣化 |
| DriftUs小 + CB_HIST急増 + CPU_MIG有 | OS要因（コンテキストスイッチ） |
| 全計測値小 | 偶発的OSジッター |
