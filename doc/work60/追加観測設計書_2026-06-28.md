### 優先度1: PageFault EWMA 閾値改善

**ステータス**: 🔄 設計変更必要
**問題**: EWMAがゼロ付近に減衰すると `delta > EWMA×5` が常に成立し偽陽性が多発（14件/16件）

**修正案**:
```
// 従来:
relative: delta > EWMA * 5

// 修正後:
relative: delta > max(EWMA * 5, 500)
```

**変更ファイル**: `Timer.cpp` MEMブロック内の閾値計算

```cpp
// 従来:
const uint64_t threshold = std::max<uint64_t>(kAbsoluteThreshold,
    static_cast<uint64_t>(thresholdRel));

// 修正後:
constexpr uint64_t kRelativeMinThreshold = 500;  // 相対条件の最小値
const uint64_t threshold = std::max<uint64_t>(kAbsoluteThreshold,
    std::max<uint64_t>(static_cast<uint64_t>(thresholdRel), kRelativeMinThreshold));
```

**効果**: EWMA=0.02 でも `threshold_rel = max(0.1, 500) = 500` となるため、Delta=1~499 では発火しない。真の異常（Delta > 50000 または Delta > EWMA×5 かつ Delta > 500）のみ警告。

### 優先度2: WorkingSet 急減検出（新規）

**目的**: OSがWorkingSetをトリミングしたタイミングを検出する。WSが突然減少した場合、OSがメモリ圧力を検知してページを解放した証拠になり、PageFault再発生の引き金になる。

**制御**: `CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS`（既存）

**出力先**: MEMブロック（既存の1秒ログに追記）

**設計**:

```cpp
// 前回のWS(MB)と比較して、5%以上減少したら警告
static uint64_t s_prevWsMB = 0;
if (s_prevWsMB > 0 && memInfo.workingSetMB < s_prevWsMB) {
    const double wsDropPct = 100.0 * (1.0 - static_cast<double>(memInfo.workingSetMB)
        / static_cast<double>(s_prevWsMB));
    if (wsDropPct > 5.0) {  // 5%以上減少
        const auto seq = convo::fetchAddAtomic(...);
        diagLog(prefix + " [Seq=" + seq + "] [MEM] WSTRIM: WS dropped "
            + juce::String(static_cast<juce::int64>(s_prevWsMB - memInfo.workingSetMB)) + "MB ("
            + juce::String(wsDropPct, 1) + "%)");
    }
}
s_prevWsMB = memInfo.workingSetMB;
```

**出力例**:
```
[22:48:54] Gen=5 Ticks=... [Seq=15] [MEM] WSTRIM: WS dropped 45MB (2.1%)
```

**抑制**: WSが5%以上減少したときのみ出力（通常時0行）。

### 優先度3: Timer callback 欠落カウンタ（新規）

**目的**: 557msのTimer jitter期間中にTimer callbackが何回欠落したかを定量化する。

**制御**: `CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS`（既存）

**出力先**: Timer jitter ブロック内に追記

**設計**:
```cpp
// intervalが expected を大きく超えた場合、欠落したcallback数を推定
if (std::abs(timerJitterMs) > jitterThreshold) {
    const int64_t missedCallbacks = static_cast<int64_t>(
        (timerIntervalMs - expectedMs) / expectedMs + 0.5);
    // 出力に missed=N を追加
    // [TIMER] jitter: interval=557ms (expected=100ms, delta=+457ms, missed=5)
}
```

**出力例**:
```
[23:00:07] Gen=5 Ticks=... [TIMER] Seq=699 jitter: interval=657.24ms (expected=100ms, delta=+557.24ms, missed=5)
```

`missed=5` により「Timerが5回スキップされた」ことが明確になる。
