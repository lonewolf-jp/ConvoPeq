## 各改修の要点

### F: diagLog Release対応（基盤整備）
diagLog() がファイル間で不統一。Timer.cpp準拠に全9ファイルを統一。
CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS で制御。

### A: コールバック受信タイムスタンプ（最重要）
OS/Driver区間のジッターを可視化。atomic<double>禁止→atomic<int64_t>。
exchangeAtomic不使用→load+store。XRUN出力に DriftUs を追記。

### G: CPU migration記録
GetCurrentProcessorNumber() でCPUコア遷移を検出。
XRUN直前にmigration→OS問題。コスト数ns〜数十ns。

### C: ConvolverProcessor全体時間
固定閾値禁止→callback予算の割合（0.2倍）で判定。
[CONV_TIME] タグで出力。

### C2: StereoConvolver単体時間
FFT畳み込みのみを分離計測。[STCONV_TIME] 出力。
[CONV_TIME]との差分でRCU管理オーバーヘッドを算出。

### D: EQ処理時間
EQ IIRフィルタ処理時間。割合閾値0.1倍。[EQ_TIME]出力。

### B: Callback総時間（条件付き）
全callback常時計測禁止。XRUN発生時のみ [CB_PROC_TIME] 出力。
startUsだけ常時取得（50ns）。

### E: ANS係数切替（優先度低）
<1us推定。優先度を降格。Step 8で実装。

## 実装順序

1. F (diagLog統一) - 基盤
2. A (callback entry) - OS/Driver区間可視化
3. G (CPU migration) - OS問題検出
4. C (Convolver全体) - MKL FFT時間
5. C2 (StereoConv個別) - FFTvsRCU分離
6. D (EQ timing) - EQ時間
7. B (条件付きCB時間) - XRUN時総時間
8. E (ANS timing) - 確認用

## 仮説判定マトリクス

| 観測パターン | 原因 |
| --- | --- |
| DriftUs大 + CPU_MIG有 | OSスケジューラ問題 |
| DriftUs大 + CPU_MIG無 | OS/Driver DPC/ISR問題 |
| DriftUs小 + CONV_TIME大 + STCONV_TIME大 | FFT畳み込み遅延 |
| DriftUs小 + CONV_TIME大 + STCONV_TIME小 | Convolver管理処理遅延 |
| DriftUs小 + EQ_TIME大 | EQ IIRフィルタ遅延 |
| DriftUs小 + CB_PROC_TIME大 + 全内部計測小 | 未計測内部要因 |
| 全計測値小 | 偶発的OSジッター |
