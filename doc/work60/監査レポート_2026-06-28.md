## 発見事項

### 🔴 要修正 #1: AudioBlock.cpp の配線漏れ（重要）

**問題**: `AudioEngine.Processing.AudioBlock.cpp` に `updateAtomicMaximum()` / `callbackCount_.fetch_add()` の呼び出しが**一切存在しない**。

**影響**: CBSUMMARY ブロック（Timer.cpp L748-777）は `intervalMaxUs_` / `callbackMaxUs_` / `callbackCount_` を読み取っているが、これらを更新するコードがないため、常に0のままで**CBSUMMARYが一度も出力されない**。

**修正方針**: `processBlock()` 内の各callback末尾に以下を追加:
```cpp
const uint32_t cbUs = static_cast<uint32_t>(callbackEndUs - callbackStartUs);
updateAtomicMaximum(callbackMaxUs_, cbUs);
const uint32_t ivUs = static_cast<uint32_t>(callbackIntervalUs);
updateAtomicMaximum(intervalMaxUs_, ivUs);
callbackCount_.fetch_add(1u, std::memory_order_relaxed);
```

### 🟡 要改善 #2: CMakeLists.txt ONEAPI_ROOT の環境変数名

**問題**: 修正前は `$ENV{%ProgramFiles(x86)%}` と不正なCMake構文を使用。修正後は `ONEAPI_ROOT` と `ProgramFiles` の2経路で検出するように変更済み。

**状態**: ✅ 修正済み

### 🟡 要改善 #3: Obsoleteログに publishedGeneration 未使用

**問題**: 計画書では `publishedGeneration_`（または相当する `lastCommittedRuntimeGeneration_`）を obsolete ログに含める設計だが、実装では `currentGen` を使用していない。現在は単に `task.generation` と `wastedMs` のみ出力。

**影響**: 低。`task.generation` と `wastedMs` だけでも目的は達成可能。`currentGen` は補足情報。

### 🟡 要改善 #4: timerPeriodMs_ の atomic 化

**問題**: `timerPeriodMs_` は `int`（非atomic）。設計上はTimerスレッドのみが読み書きするため問題ないが、将来複数スレッドからの読み取りが必要になった場合に競合のリスク。

**影響**: 低。現状Timerスレッドのみがアクセス。

## 結論

実装は計画書の設計と概ね一致しているが、**致命的な配線漏れが1件**（CBSUMMARYの入力元となるAudioBlock.cppの更新コード未実装）および**軽微な改善点が2件**存在する。

最も優先度が高いのは **AudioBlock.cppへの `updateAtomicMaximum()` 呼び出し追加** であり、これがないとCBSUMMARYが全く機能しない。
