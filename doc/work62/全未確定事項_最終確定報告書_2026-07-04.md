## 4. DiagDrain 実装のための確定設計情報

### 4.1 挿入位置（確定）

```
src/audioengine/AudioEngine.Timer.cpp
  timerCallback() 末尾
    ├── s_timerExecStartMs 更新 (L1100)
    ├── DiagDrain ブロック     ← ★ flushLogBuffer() の直前に挿入
    ├── flushLogBuffer()    (L1104)
    └── #endif // CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS (L1105)
```

### 4.2 アクセス可能な変数（確定）

| 変数 | 型 | アクセス方法 |
|------|-----|------------|
| `diagBuffer` | `LockFreeRingBuffer<DiagEvent, 512>` | AudioEngine メンバ（`this->diagBuffer`） |
| `diagTickPushed/Popped/Dropped` | `DiagPerTickCounter` | `rtAuxMutable_.diagTickPushed` |
| `diagTotalPushed/Popped` | `std::atomic<uint64_t>` | `rtAuxMutable_.diagTotalPushed` |
| `runtimeWorld` | `RuntimeWorld*` | timerCallback 内ローカル変数 |
| `MaxDrainPerTick` | `16` | `DiagRuntimeLimits::MaxDrainPerTick`（ソースコードの実値） |

### 4.3 DiagDrain 実装方針（確定）

- **全8カテゴリを一度に実装する**（後からの追加はコンフリクトリスクが高いため）
- カテゴリ: CPU_MIG, CallbackArrival, CallbackSequence, DspTiming, CallbackStage, EqTime, ConvTime, StereoConvTime
- 各カテゴリのフォーマット文字列は work60 設計書 `診断ログRT違反_numeric-only設計レポート_2026-06-30.md` の Phase 8 に従う
- `MaxDrainPerTick = 16` の制限を守る（全push 65 events/sec << drain 160 events/sec）

### 4.4 正常時サンプリングの方針（確定）

- 全 tick の CallbackArrival 出力は **行わない**（diagBuffer 512エントリ飽和リスク）
- DiagDrain 側で `(event.eventIndex & 0xF) == 0` の 1/16 サンプリングフィルタを適用
- 残り 15/16 は pop するが出力しない（リングバッファの滞留防止）

### 4.5 sizeof(DiagEvent) 確認手順（確定）

1. `AudioEngine.h` L451 の `// template struct DiagEventSizeCheck<sizeof(DiagEvent)>;` のコメントを外す
2. Release ビルド → リンカエラーに `DiagEventSizeCheck<XX>` と表示される
3. その XX の値で L456 の `static_assert` を更新
4. L451 の行を再コメントアウト
