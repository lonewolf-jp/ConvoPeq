## 1. ゲイン解析の重要発見

`gain_staging_analysis_2026-06-21.md` から以下の重要事実が判明した：

### 1.1 IR Scale Factor による自動正規化

```cpp
// IRConverter.cpp: computeScaleFactor()
makeup = 1.0 / sqrt(maxChannelEnergy);
scaleFactor = makeup × 0.5011872336272722;  // さらに -6dB 安全マージン
```

IR は **エネルギー正規化 + 安全マージン -6dB** される。Convolver 出力が入力を超えることは典型的には起こらない。

### 1.2 モード別ゲインの整合性

| モード | inputHeadroom | IR scaleFactor | outputMakeup | 正味(SoftClip前) |
|:-----:|:------------:|:--------------:|:------------:|:---------------:|
| PEQ-only | 0dB | — | 0dB | **EQ出力 × 1.0** |
| Conv→Peq | -6dB | -6dB | +12dB | **EQ出力 × 1.0** |

```
PEQ-only:   0dBFS入力 → [×1.0] → [EQ] → ... → [×0dB] → SC → NS
Conv→Peq:   0dBFS入力 → [×0.5] → [Conv(IR×0.5)] → [EQ] → ... → [×3.98] → SC → NS
                                           ↓
           Conv出力 = 入力 × 0.5 × 0.5 = 0.25 → EQ後 → ×3.98 = EQ出力 × 1.0
```

**SoftClip直前の信号レベルは両モードでほぼ同等** となる。これはゲイン解析の核心的発見である。

### 1.3 重要な帰結

「NS 入力が Conv→Peq で過大になる」という v5 までの仮説は **否定された**。

レベルが同等なら、原因は **信号の質的な違い**（波形形状・周波数分布・位相関係など）にあるはずである。

## 2. 残る差異（Conv→Peq と PEQ-only の違い）

レベルの差が否定された今、原因は以下の差異に絞られる：

### 2.1 Convolver による波形変化

| 項目 | PEQ-only | Conv→Peq |
|:----:|:--------:|:---------:|
| 信号の起源 | 入力そのもの | 入力 × IR（畳み込み） |
| 残響成分 | なし | IRのテール（残響） |
| 位相特性 | EQのみ | EQ + IR の位相 |
| Crest Factor | 入力依存 | IR依存（残響で平均化） |
| 低域の時間分布 | 入力波形通り | IRテールで引き伸ばされる |

### 2.2 低域特化性の説明

低域（40-80Hz）がトリガーとなる理由：

```
低域信号（周期12.5-25ms）
  → IRのテールで低域エネルギーが時間的に引き伸ばされる
  → 典型的なブロックサイズ（512samples @48kHz = 10.6ms）では
     1周期未満の信号でも IR の影響でブロック間でエネルギーが持続
  → SoftClip での非対称クリップの発生確率が上昇
  → クリップの非対称成分（偶数次高調波）が発生
  → これらの成分が NS で処理され、ノイズとして知覚
```

### 2.3 SoftClip の非対称性

SoftClip の asymmetry パラメータ：

```cpp
clipAsymmetry = 0.10 * sat;
```

デフォルト sat=0.1 で asymmetry = 0.01。非対称性は小さい。しかし、レベルが同等でも **IRによる波形の非対称化**（Convolver出力の偏り）により、SoftClip で非対称クリップが発生しやすくなる可能性がある。

## 3. 「NS内部状態不安定化」仮説の再評価

### 3.1 NS への入力を再計算

SoftClip 直前の信号レベルが両モードで同等の場合、NS 入力も同等となる：

```
SoftClip出力 (sat=0.1) ≲ 0.905（threshold）
↓ × kOutputHeadroom (0.891)
NS 入力 ≈ 0.806
NS quantize 出力 ≈ 0.806（通常動作範囲内）
↓
post-NS hard clamp ±0.891 → 通過（0.806 < 0.891）
```

**通常時は post-NS clamp に引っかからない。**

### 3.2 「NS内部状態」説の困難

レベルが同等なら NS 内部状態も同等になるはず。レベル差が原因なら説明がつくが、ゲイン解析でレベル差が否定されたため、この仮説は大幅に弱まる。

## 5. 推奨診断テスト（改訂版）

ゲイン解析により「NS入力過大説」が否定されたため、診断の優先順位を変更する。

### テスト1（最重要）: Output Makeup 0dB

`newOutputMakeupDb = 12.0 → 0.0`

**消える→SoftClip以降の経路に関係。消えない→ConvolverまたはIRが原因。**

### テスト2: SoftClip完全OFF

`softClipEnabled = false`

**消える→SoftClipの非線形性が原因。消えない→SoftClip以前の経路が原因。**

### テスト3: OS=1x固定

`manualOversamplingFactor = 1`

**消える→Downsamplerが原因の一部。**

### テスト4: NoiseShaper完全OFF

`applyDither = false`

**消える→NSが原因。消えない→NS以前が原因。**

### テスト5: 異なるIRで比較

短いIR（数十msのスナップ）と長いIR（数秒のリバーブ）で比較。IRの長さで症状が変わるか確認。

## 6. 結論

| 項目 | 判定 |
|:----|:----|
| +12dB makeup の存在 | ✅ 確定 |
| IR Scale Factor による正規化 | ✅ 確定（新発見） |
| NS入力レベル差（Conv→Peq vs PEQ-only） | ❌ **否定**（v5までの仮説が崩壊） |
| post-NS clamp 主犯説 | ❌ 否定（v4） |
| **最有力: IR波形特性 + SoftClip相互作用** | **推定（テストで検証可能）** |
| 現時点の不足データ | IR毎の症状再現性、OS=1xでの挙動 |

**ゲイン解析の最大の成果**: NS入力レベルが両モードで同等であることが判明したため、「過大入力→NS暴走」という単純な説明は成立しない。原因はより微妙な**信号の質的差異**にある。
