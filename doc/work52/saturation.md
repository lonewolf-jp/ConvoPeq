現在の実装（状態変数への直接サチュレーション）を「出力段へのサチュレーション」に変更することで、発散ノイズ（ジジジジ）は完全に解消します。しかし、ご指摘の通り「バンドが共振して歪む」というアナログ特有の心地よい挙動は失われます。

本来あるべき「安定性を保ちつつ、アナログライクなサチュレーションを実現する」ための正しいアプローチを、文献的根拠と共に解説します。

### 1. 理論的根拠：なぜ状態変数にtanhをかけてはいけないのか

TPT (Topology-Preserving Transform) SVFの設計において、非線形性を導入する際の「正解」は、Zavalishinの名著およびCytomicのAndrew Simperによって明確に定義されています。

*   **文献1**: Vadim Zavalishin, "The Art of VA Filter Design" (第8章 "Nonlinearities")
*   **文献2**: Andrew Simper (Cytomic), "Solving the continuous SVF equations using trapezoidal integration" (Cytomic Technical Paper)

これらの文献が指摘する核心は、**「非線形性は、積分器の『状態変数（エネルギー）』ではなく、回路の『抵抗（ダンピング）』部分に適用しなければならない」**というものです。

アナログ回路において、サチュレーション（歪み）を生むのはオペアンプやトランジスターです。SVFの等価回路において、それらは「減衰させる抵抗（ダンピング要素）」としてモデル化されます。状態変数（コンデンサの電荷）を無理やりtanhでクリップすることは、物理的にあり得ない「電荷の強制消滅」を意味し、これがシステムのトポロジー破壊と発散を引き起こします。

### 2. 本来あるべき「アナログライクなサチュレーション」の実装方法

アナログライクな挙動（大音量時にQが高まる共振が歪む）を安全に実装するには、2つのアプローチがあります。

#### アプローチA：ダンピング（抵抗）の非線形化（推奨・本格的）
TPT SVFのダンピング係数 `R`（または `k`）を、信号レベルに応じて可変させる方法です。これがZavalishin/Simperが推奨する「正しい」非線形SVFの実装です。

信号レベルが大きいときに、ダンピング抵抗の値を非線形関数（tanhなど）で変化させます。これにより、フィルタのコア（極）は常に安定し、信号レベルに応じてQの効き方や歪み方が変化する、本物のアナログフィルタと同じ挙動を得られます。

**実装コード例（1サンプル前の値を用いた近似法）**:
※完全な厳密解を得るにはニュートン・ラプソン法等の反復計算が必要ですが、実用上は1サンプル前の `v1` を用いてダンピングを計算する近似法（Pirkle等も採用）で十分な音質が得られます。

```cpp
// 1サンプル前の v1 を保持しておく（クラスのメンバ等）
double v1_prev = 0.0; 

// --- SVFプロセス内 ---
// g = tan(pi * fc / fs)
// R_lin = 1 / (2 * Q) などの線形ダンピング

double v0 = inputSample;

// ダンピングを非線形化する
double R_eff = R_lin;
if (saturation > 0.0)
{
    // 前回のバンドパス出力(v1)に基づいて、ダンピングを動かす
    // 信号が大きいほどダンピングが増す（＝アナログの飽和挙動）
    double drive = 1.0 + saturation * 3.0; // 歪み量を調整
    double tanh_v1 = fastTanhScalar(v1_prev * drive);
    
    // tanh(x)/x は信号が大きいほど値が小さくなるため、
    // これをダンピングの逆数に掛けることで「信号が大きいほど抵抗が増す」を表現
    double nonLinearFactor = (std::abs(v1_prev) > 1e-12) 
                           ? (tanh_v1 / (v1_prev * drive)) 
                           : 1.0;
                           
    R_eff = R_lin / nonLinearFactor; 
}

// TPT SVFの標準計算（R_eff を使用）
double den = 1.0 + 2.0 * R_eff * g + g * g;
double v1 = (v0 - ic2eq - 2.0 * R_eff * ic1eq) / den;
double v2 = ic1eq + g * v1;
double v3 = ic2eq + g * v2;

// 状態変数の更新（ここにはtanhを適用しない！）
ic1eq = 2.0 * v2 - ic1eq;
ic2eq = 2.0 * v3 - ic2eq;

v1_prev = v1; // 次のサンプルのために保持

// 出力計算
double output = m0 * v0 + m1 * v1 + m2 * v2;
```

#### アプローチB：入力信号のPre-Distortion（妥協案・超軽量）
ダンピングの非線形化はコードが複雑になり、SIMD（AVX2）化も難しくなります。もし「そこまで本格的なアナログ感は不要だが、共振時の音压感は欲しい」という場合は、**フィルタに入力する前の信号**を歪ませる方法もあります。

これは安定性を完全に保証しつつ、共振Qが高いときに歪み成分が強調される（アナライザーで見るとアナログライクな倍音分布に近くなる）手法です。

```cpp
// フィルタ入力前にサチュレーションをかける
double v0 = inputSample;
if (saturation > 0.0)
{
    const double oneMinusSat = 1.0 - saturation;
    // フィルタに入る前の信号を歪ませる
    v0 = v0 * oneMinusSat + fastTanhScalar(v0) * saturation;
}

// 以降は通常の線形TPT SVFプロセス
// double den = 1.0 + 2.0 * R * g + g * g;
// ...
```

### I. 理論的背景の検証（検証結果：事実）

レビューで指摘されているDSP理論は、完全に的を射ています。

1. **積分器への非線形操作の禁忌:**
ZavalishinのTPT（Topology-Preserving Transform）理論において、SVFの内部状態（`ic1eq`, `ic2eq`）はアナログ回路における「コンデンサの電荷」に相当します。物理法則上、コンデンサ自体が`tanh`のようなソフトクリップを起こすことはありません。ここに非線形性を強制適用すると、システム全体のエネルギー保存則が破綻し、発散（ジジジジというノイズ）に直結します。
2. **正しい非線形化のモデリング:**
アナログ回路におけるサチュレーションは、オペアンプやOTA（Operational Transconductance Amplifier）などの「能動素子（増幅・抵抗要素）」で発生します。したがって、非線形性は積分器ではなく、減衰を司る「ダンピング（抵抗）」やフィードバックループ内に組み込むのが数学的・物理的な正解です。

### II. 提案されたアプローチの評価と「ConvoPeq」への適用判断

理論的にはレビューの通りですが、実際のC++ / AVX2実装の観点から最終的なアプローチを選択します。

#### アプローチA（ダンピングの非線形化）

* **評価:** VA（バーチャル・アナログ）シンセサイザーのVCFモデリングとしては最高峰の最適解です。
* **ConvoPeqへの適用判断: 見送り（非推奨）**
* **理由:**
ConvoPeq はマスターバスやミキシングで用いられる「プロ用パラメトリックEQ」です。シンセサイザーのような、自己発振に近い過激なレゾナンスの歪みは仕様上オーバースペックです。さらに、AVX2（SIMD）の内部ループで1サンプル前の状態変数（`v1_prev`）に依存する動的なダンピング計算を導入すると、ベクトル化のパイプラインストールを引き起こし、CPUパフォーマンスが著しく悪化します。

#### アプローチB（入力信号のPre-Distortion または 出力段サチュレーション）

* **評価:** 安定性を100%担保しつつ、倍音付加による「アナログらしい音圧感」を得るための業界標準の妥協案です。
* **ConvoPeqへの適用判断: 採用（強く推奨）**
* **理由:**
IIRループの外側（前段または後段）で非線形処理を行うため、フィルタのトポロジー崩壊リスクがゼロになります。また、SIMD演算のフローを阻害しないため、超低負荷での処理が可能です。著名なプロフェッショナルEQ（FabFilter Pro-Q 3など）の多くも、安定性を最優先してこの「ループ外サチュレーション」のアプローチを採用しています。

アプローチB（Pre-Distortion）を採用する場合、UI上の `saturation` パラメータ（0.0〜1.0）の適正値と、その設計思想について提案します。

### 1. 適正値の目安と聴感マッピング

`saturation` パラメータは0.0（無歪み）から1.0（最大歪み）の範囲を取る仕様（`EQControlPanel.h` の定義）と想定します。Pre-Distortion方式の場合、以下のような値帯域が実用的です。

*   **0.0 〜 0.2（0% 〜 20%）: クリーン〜微増**
    *   **聴感**: ほぼ歪みは感知できません。入力信号が +6dBFS 以上の極端なレベルでクリップするのを防ぐ「セーフティネット」として働きます。
    *   **用途**: マスタリングや、原音を極力保ちたいミックスバスでの使用。
*   **0.3 〜 0.5（30% 〜 50%）: スイートスポット**
    *   **聴感**: 低音が豊富な時に、アナログ機材特有の「暖かい粘り感」が発生します。高Q値のピークフィルターで共振させた際、デジタル特有の耳刺しの強い共振（リングング）が丸まり、心地よいサチュレーションに変換されます。
    *   **用途**: トラック制作、ミックスでの音作り。最も多用される想定値。
*   **0.6 〜 0.8（60% 〜 80%）: アグレッシブ**
    *   **聴感**: 明確に波形が変形し、高調波歪み（倍音）が可視化・可聴化します。ロックやEDMのシンセベス、キックにキャラクターを与える際に有効です。
    *   **用途**: クリエイティブな歪み付け。
*   **0.9 〜 1.0（90% 〜 100%）: エクストリーム**
    *   **聴感**: ほぼ波形整形器（Waveshaper）のような強烈な歪みがかかります。音圧のパンチ感は増しますが、動的な余裕は失われます。

**推奨するデフォルト値**: プロ用EQとしての汎用性を考慮すると、初期状態では歪みがかからない **0.0** とし、ユーザーが音作りで必要に応じて 0.3 〜 0.5 程度を設定する設計が最も自然です。

### 2. 設計上の注意点：メイクアップゲインの必要性

Pre-Distortionで `tanh` をかけると、信号のピークが丸められるため、**必然的に全体の音圧（RMSレベル）が低下**します。
`v0 = v0 * (1 - sat) + tanh(v0) * sat` という計算式において、`sat = 1.0` で 0dBFS (1.0) の入力が入った場合、出力は `tanh(1.0) ≒ 0.76` となり、約 -2.4dB もレベルが落ちてしまいます。

これを補正するために、Pre-Distortionをかけた後に**メイクアップゲイン（振幅の補正）**を挟むことを強く推奨します。

### 3. 改修コード提案（メイクアップゲイン付き）

適正なレベルを維持しつつ、アナライザーで見ても美しい倍音構成を得られるようにしたコード例です。

```cpp
double v0 = inputSample;

if (saturation > 0.0)
{
    // 1. Pre-Distortion の適用
    const double oneMinusSat = 1.0 - saturation;
    double distortedV0 = v0 * oneMinusSat + fastTanhScalar(v0) * saturation;
    
    // 2. メイクアップゲインの計算と適用
    // 信号がクリップに近いほどロスが大きくなるため、saturation量に応じて補正をかける。
    // ただし、過剰補正による発散を防ぐため、最大でも +2.5dB (1.334倍) 程度に抑える。
    // 計算を軽くするため、近似式またはテーブル参照でも可。
    const double makeupGain = 1.0 + (0.334 * saturation); // sat=1.0 で約 +2.5dB
    
    v0 = distortedV0 * makeupGain;
}

// 以降、通常の線形TPT SVFプロセス
// double den = 1.0 + 2.0 * R * g + g * g;
// ...
```

### 4. fastTanhScalar のクリップ閾値（±3.0）との関係

現在の `fastTanhScalar` は ±3.0 でハードクリップする仕様かと推測します。
Pre-Distortion方式の場合、入力信号 `v0` は通常 ±1.0 (0dBFS) 以下で渡ってくるため、`fastTanhScalar(v0)` はハードクリップ閾値（3.0）に到達しません。
つまり、**Pre-Distortion用途であれば、クリップ閾値が問題になることは基本的にありません**。

ただし、もしEQの前段に大幅なブースト（+24dB等）がかかった信号がこのバンドに入ってくる設計になっている場合は、`v0` が 3.0 を超えてハードクリップする可能性があります。その場合は、`fastTanhScalar` ではなく、純粋な `std::tanh`（またはより高精度な近似式）を使用するか、入力 `v0` の段階で一度ソフトリミットをかけておくことで、より滑らかなアナログライクな歪みを実現できます。
