# ISR バグ精査結果（2026-05-23）

- 対象: ユーザー提示のバグ候補一覧（重複含む）
- 方針: **Immutable Snapshot Runtime 構造を厳守**し、実コード行を根拠に妥当性を判定
- 判定基準:
  - 重大度（重大 / 高 / 中）
  - 再現可能性
  - RT安全性 / Snapshot整合性 / 音質影響

---

## 1. 結論サマリ（重複統合後）

- ユニーク論点: 約12件
- 妥当（成立）: 11件
- 条件付き（設計判断要）: 1件（mix=0時の畳み込みスキップ方針）

優先度順:

1. Snapshot単一起点化（N1/N2）
2. double経路の非対称解消（ガード/フェード/OSバイパス）
3. 異常系防御（0ch/null, numSamples）
4. Convolver負荷最適化（仕様明確化前提）

## 2. 個別判定

### A. 重大/音質: double + Oversampling のバイパスフェードで Dry未合成

- 判定: **妥当（成立）**
- 根拠:
  - `src/audioengine/AudioEngine.Processing.DSPCoreDouble.cpp:525-541`
  - OS>1分岐で `gWet` のみ適用（`gDry` 合成なし）
  - 同ファイル `:503-521`（OS=1系）には Dry/Wet 合成あり
- 影響: バイパス遷移時にレベル不連続（無音側への落ち込み）
- 修正方向（ISR準拠）:
  - OS>1分岐でも `dryBypassBufferDouble*` を使い Dry/Wet 合成に統一

### B. 重大/RT安全: doubleコールバック経路の shutdown/lifecycle/firewall 欠落

- 判定: **妥当（成立）**
- 根拠:
  - あり: `src/audioengine/AudioEngine.Processing.AudioBlock.cpp:18-47`
  - なし: `src/audioengine/AudioEngine.Processing.BlockDouble.cpp:15-23`
- 影響: 終了競合・RT検知漏れ
- 修正方向:
  - `processBlockDouble` に float経路同等の runtime scope（shutdown/lifecycle/firewall）追加

### C. 重大/immutable: 1ブロック内で snapshot 二重取得（世代混在）

- 判定: **妥当（成立）**
- 根拠:
  - `AudioBlock.cpp:154` で `kAudioEpochReaderIndex`
  - `DSPCoreFloat.cpp:191`, `DSPCoreDouble.cpp:394` で `ownerEngine->observeCurrentRuntime()`
  - `AudioEngine.h:819` より `observeCurrentRuntime()` は `kControlEpochReaderIndex`
- 影響: 同一ブロックで `ProcessingState` と EQ係数参照世代が不一致化
- 修正方向（最重要）:
  - **single-snapshot-per-block** を徹底（入口で取得・下流へ注入）
  - DSPCore内再取得を廃止

### D. 高/immutable遷移品質: double経路で snapshot fade が実質未反映

- 判定: **妥当（成立）**
- 根拠:
  - `BlockDouble.cpp:64-68` で `updateAudioThreadSnapshotFade` のみ
  - float側 (`AudioBlock.cpp:178-179`) にある `processWithSnapshot(old/new)` 相当がない
- 影響: double時のみ遷移段差化リスク
- 修正方向:
  - double経路にも old/new snapshot 処理とクロスフェード導入

### E. 高/RT規約: Convolver Audio Thread 内 `std::floor`

- 判定: **妥当（成立）**
- 根拠: `src/convolver/ConvolverProcessor.Runtime.cpp:337`
- 影響: RTでのlibm回避規約に抵触
- 修正方向:
  - libm依存しない位相処理（整数/固定演算または既存no-libm近似）に置換

### F. 中/状態機械: EQバイパス遷移が dryCopy 不成立時に進行停止し得る

- 判定: **妥当（成立）**
- 根拠:
  - dryCopy確保: `EQProcessor.Processing.cpp:543-553`
  - 遷移進行が `dryCopyBase != nullptr` ブロック依存: `:851-876`
- 影響: フェード完了せず `rtBypassedShadow` 更新停止リスク
- 修正方向:
  - フェード進行と状態更新を dryCopy有無から分離

### G. 重大/RT性能: mix=0でも畳み込み実行

- 判定: **条件付き（設計判断要）**
- 根拠:
  - `needsConvolution` 算出: `ConvolverProcessor.Runtime.cpp:294`
  - `conv->process(...)` 常時実行: `:587`
- 影響: Dry-only時に不要負荷
- 注意:
  - 単純スキップは内部状態進行を止める可能性
- 修正方向:
  - 「完全スキップ or 軽量prewarm」の仕様決定後に実装

### H. 重大/immutable: snapshot fadeで old/new が同一DSP実体になりうる

- 判定: **妥当（成立）**
- 根拠: `src/audioengine/AudioEngine.Processing.Snapshot.cpp:21-29`
- 影響: old/new独立性崩壊
- 修正方向:
  - fading未解決時に activeへフォールバックしない（dry-oldまたはfade無効）

### I. 高/遷移品質: snapshot専用経路で sample-rate 不整合ガード欠落

- 判定: **妥当（成立）**
- 根拠:
  - 通常経路ガードあり: `AudioBlock.cpp:141-147`
  - snapshot経路ガードなし: `Snapshot.cpp:7-43`
- 修正方向:
  - `processWithSnapshot` に同等ガード追加

### J. 高/クラッシュ: 0ch時 null 参照リスク（出力処理）

- 判定: **妥当（成立）**
- 根拠:
  - float: `DSPCoreIO.cpp:312,318,435`
  - double: `DSPCoreDouble.cpp:584,608,650,678,707,715`
- 修正方向:
  - `numChannels<=0` 早期return統一
  - `dataL==nullptr` 以降の処理防止

### K. 中〜高/寿命整合: snapshot処理で activeNode 直参照

- 判定: **妥当（成立）**
- 根拠:
  - `Snapshot.cpp:23-29` 直参照
  - float通常経路は `dspHandleRuntime_.resolve(...)` 併用（`AudioBlock.cpp:94-97`）
- 修正方向:
  - snapshot経路も resolver統一

### L. 中/防御不備: `numSamples <= 0` 分岐で clear に非正値

- 判定: **妥当（成立）**
- 根拠:
  - `EQProcessor.Processing.cpp:523-527`, `:899-903`
- 修正方向:
  - `numSamples < 1` は即return
  - clear呼び出しは `numSamples > 0` 前提に限定

---

## 3. Immutable Snapshot Runtime 厳守の実装修正ルール

1. Snapshot取得は **Audio block入口で1回のみ**
2. Audio Thread は `kAudioEpochReaderIndex` のみ使用
3. DSPCore内で `observeCurrentRuntime()` 再取得禁止
4. snapshot fade で old/new 同一実体を禁止（不変条件化）
5. フェード進行はバッファ確保可否から独立（状態機械停止を防止）

---

## 4. 推奨実装順

1. N1/N2（snapshot単一起点化 + reader index是正）
2. double経路対称化（ガード導入 + snapshot fade + OS bypass修正）
3. 異常系防御（0ch/null, numSamples）
4. Convolver最適化（仕様明文化後に適用）
