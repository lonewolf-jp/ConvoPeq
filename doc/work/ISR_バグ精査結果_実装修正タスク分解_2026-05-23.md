## 目的

監査で成立した不具合（重大〜中）を、**Immutable Snapshot Runtime（ISR）構造を厳守**しつつ、
実装順と依存関係を明示したタスクへ分解する。

本計画の優先指標:

1. 1ブロック1スナップショット整合性の回復（N1/N2）
2. double経路の float対称化（安全ガード/遷移品質）
3. RT安全・異常系クラッシュ防止
4. 性能改善は仕様固定後に適用

---

## 前提ルール（本タスク全体）

- vendor配下は編集禁止: `JUCE/`, `r8brain-free-src/`
- Audio Thread でブロッキング/lock/libm/動的確保を導入しない
- Snapshot取得は Audio block 入口で1回のみ
- Audio Thread は `kAudioEpochReaderIndex` のみ使用
- DSPCore内から `observeCurrentRuntime()` を再呼び出ししない

### ISR不変条件（Invariant）

- `snapshotFrom` / `snapshotTo` の参照は同一ブロック内で固定し、途中で再取得しない。
- snapshot fade 実行時は `snapshotTo != nullptr` を必須とし、`snapshotFrom == nullptr` の場合は明示フォールバック（dry-old または fade無効）を適用する。
- old/new DSP は同一実体を禁止し、`old == new` 検出時は assert + 安全フォールバックへ遷移する。
- Audio Thread 経路では control reader index の使用を禁止する。
