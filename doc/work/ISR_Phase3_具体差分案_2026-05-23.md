- 変更対象: Convolver runtime の関数群（遅延読み出し正規化）

## 2. ファイル別 具体差分案

## 2.1 `ConvolverProcessor.Runtime.cpp`（no-libm 正規化置換）

### 編集ポイント（ConvolverNoLibmNorm）

`std::floor` ベースの位置正規化を、no-libm の整数演算中心実装へ置換する。

### 具体手順（ConvolverNoLibmNorm）

1. `std::floor(x)` を使う箇所を特定し、
   - 整数部 `i`
   - 小数部 `f`
   分解処理を no-libm 代替へ差し替える。
2. 負値入力を含むため、単純キャストでなく「数学的 floor と等価」になる分岐を実装。
3. 正規化後インデックスを ring 範囲へ wrap（`[0, N)`）する。
4. `f` は `0.0 <= f < 1.0` を満たすよう補正し、補間係数として利用。

### 差分イメージ（ConvolverNoLibmNorm）

- Before:
  - `std::floor` に依存して整数部/小数部を算出。
- After:
  - 整数演算＋分岐で同等の `i/f` を生成し、libm 呼び出しを除去。

### DoD直結チェック（ConvolverNoLibmNorm）

- Audio Thread 実行経路に `std::floor` が残らない。
- `f` の範囲が常に `[0, 1)` に収まる。

---

### 編集ポイント（ConvolverBoundaryParity）

境界ケース（wrap/負値/frac近傍）で置換前後の挙動差を抑える。

### 具体手順（ConvolverBoundaryParity）

1. wrap ケース:
   - 末尾付近 `N-ε` から `0+ε` へ跨るとき、インデックス飛びと係数不連続を防止。
2. 負値ケース:
   - `x < 0` 入力で数学的 floor 相当の整数部を返すことを確認。
3. frac 近傍ケース:
   - `f≈0` / `f≈1` で係数の不連続を起こさないよう、補正順序を固定。
4. 既存補間式（linear等）への入力契約を維持する。

### 差分イメージ（ConvolverBoundaryParity）

- Before:
  - libm floor に依存した境界安定性。
- After:
  - no-libm でも境界動作が同等（クリック・段差を増やさない）。

### DoD直結チェック（ConvolverBoundaryParity）

- wrap でインデックス破綻がない。
- 負値/frac近傍で出力連続性が悪化しない。

---

### 編集ポイント（ConvolverRTContract）

置換実装が RT制約を満たすことを保証する。

### 具体手順（ConvolverRTContract）

1. no-libm 置換に伴い、
   - lock
   - 動的確保
   - 追加I/O
   を導入していないことを確認。
2. 補助ヘルパを追加する場合は `constexpr`/`inline`/POD中心で最小化。
3. 処理量増加を避けるため、分岐数とメモリアクセスを抑制。

### DoD直結チェック（ConvolverRTContract）

- RT規約違反（libm/lock/alloc/blocking）が新規に発生しない。

## 5. 実装メモ（レビューで見られる点）

- Phase 3 は「libm排除」に限定し、Phase 4（性能最適化）を混在させない。
- floor互換実装で最も壊れやすいのは `x<0` と `f≈1` 境界なので、最優先で検証する。
- 既存補間式を変更しない（入力正規化のみ差し替える）ことで回帰範囲を最小化する。
