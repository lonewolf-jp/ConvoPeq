## 1. プロジェクト概要

BarcelonaReverbera は、sbrk devices が開発している **Non-Uniform Partitioned Convolution Reverb VST3 プラグイン**（C++ / JUCE 8）で、バルセロナの대표적인空間を_convolution reverb_用インパルス応答（IR）として提供する目的を持つ。ライセンスは Custom Non-Commercial。

**README からの主要記述**：
- 非一様分割畳み込み（Non-Uniform Partitioned Convolution）を実装
- ソースコードは `src/BarcelonaReverbera` 配下
- テンプレートメタプログラミングにより複数の FFT ステージをコンパイル時に自動生成
- FFT ライブラリに **PFFFT**（bitbucket.org/jpommier/pffft）を使用
- IR の生成には `ConvertWavsToCArray/ConvertWavsToCArray.py`（WAV → C++ ヘッダ変換）を利用

## 6. IrBuffers（IR データ管理）

`IrBuffersAutoGenerated.h` に自動生成される。**手動編集禁止**（`#pragma once` + 先頭に注意書き）。

### 6.1 生成元の Python スクリプト

`ConvertWavsToCArray/ConvertWavsToCArray.py` の処理内容：

1. **入力**: ステレオ WAV ファイル（48 kHz、3×LARGEST_STAGE_SAMPLES 以上、10秒以下）
2. **末尾ゼロ除去**: 各 WAV の末尾の連続ゼロをトリム
3. **ゼロパディング追加**: 各バッファ末尾に `m_irBuffersExtraZeros`（= 16384）サンプルのゼロを付与
4. **出力**: `IrBuffersAutoGenerated.h`（C++ ヘッダファイル）

```python
# 出力形式（1つのIRバッファあたり）
float m_audioBuffer_N[2][audioLen + 16384] =
{
    { ch0_s0, ch0_s1, ..., ch0_s(audioLen-1), 0.0, ..., 0.0 /* 16384個 */ },
    { ch1_s0, ch1_s1, ..., ch1_s(audioLen-1), 0.0, ..., 0.0 /* 16384個 */ }
};
```

## 7. ConvolutionEngine（コア畳み込みエンジン）

### 7.1 非一様分割スキーム

`ConvolutionEngine.h` の先頭に記されたスキーム（BS = `BCNRVRB_SMALLEST_STAGE_SIZE` = 64）：

```
Method:          DIRECT   BS     2*BS    4*BS     8*BS     ...   FFT
Blocks:          H0-H1   H2-H3  H4-H7  H8-H15  H16-H31  ...
Audio offset:    X-1,X0  X-2,X3  X-4-7  X-8-15  X-16-31  ...
```

- **Direct Stage**（H0〜H1）：時間領域直接畳み込み。レイテンシ 0。
- **FFT Stage BS**（H2〜H3）：ブロックサイズ = 64。全ブロック数 = 2。
- **FFT Stage 2×BS**（H4〜H7）：ブロックサイズ = 128。全ブロック数 = 2。
- **FFT Stage 4×BS**（H8〜H15）：ブロックサイズ = 256。全ブロック数 = 2。
- 以降、最大 `BCNRVRB_LONGEST_STAGE_SIZE`（16384）までの各ブロックサイズで FFT ステージが存在。
- **LONGEST ステージ**：`LONGEST_STAGE_BLOCK_COUNT`（= `(IR_LEN/LONGEST)+1-2`）個のブロックを処理。

### 7.2 テンプレートメタプログラミングによるステージ生成

```cpp
// GenerateFftStages: 再帰テンプレートで倍増ブロックサイズのステージを生成
template<uint32_t BlockSize, uint32_t LastBlockSize, bool ReplacesFirstStage, typename... Stages>
struct GenerateFftStages {
    using type = typename GenerateFftStages<
        BlockSize * 2, LastBlockSize, ReplacesFirstStage,
        Stages..., ConvolutionEngineFftStage<BlockSize, 2, ReplacesFirstStage>
    >::type;
};

// 終端特殊化（BlockSize == LastBlockSize で再帰停止）
template<uint32_t LastBlockSize, bool ReplacesFirstStage, typename... Stages>
struct GenerateFftStages<LastBlockSize, LastBlockSize, ReplacesFirstStage, Stages...> {
    using type = std::tuple<Stages..., ConvolutionEngineFftStage<LastBlockSize, ...>>;
};
```

これにより以下がコンパイル時に生成される（`BCNRVRB_SMALLEST_STAGE_SIZE=64`, `LONGEST_STAGE_SIZE=16384` の場合）：

```cpp
std::tuple<
    ConvolutionEngineFftStage<64,   2>,    // BS
    ConvolutionEngineFftStage<128,  2>,    // 2*BS
    ConvolutionEngineFftStage<256,  2>,    // 4*BS
    ConvolutionEngineFftStage<512,  2>,    // 8*BS
    ConvolutionEngineFftStage<1024, 2>,    // 16*BS
    ConvolutionEngineFftStage<2048, 2>,    // 32*BS
    ConvolutionEngineFftStage<4096, 2>,    // 64*BS
    ConvolutionEngineFftStage<8192, 2>,    // 128*BS
    ConvolutionEngineFftStage<16384, LONGEST_STAGE_BLOCK_COUNT>
>
```

### 7.3 ConvolutionEngineDirectStage（時間領域直接畳み込み）

**目的**: IR の最初の2ブロック（H0, H1）を時間領域で畳み込み、ゼロレイテンシを実現。

```cpp
template<uint32_t _maxBlockSize>
class ConvolutionEngineDirectStage
{
    // m_convAccum[2][2 * m_maxBlockSize]  // 循環アキュムレータ（16byte align）
    // m_ir[2][2]                           // ダブルバッファ IR ポインタ（2ch×2バッファ）
    // std::atomic<uint8_t> m_irIndex       // ロックフリー IR 切替
    // m_currentPos                          // 循環バッファ書き出し位置

    // process():
    //   for each input sample i:
    //     for j in [0, 2*blockSize):
    //       m_convAccum[ch][(currentPos+j) % (2*blockSize)] += in[ch][i] * ir[ch][j]
    //     out[ch][i] += m_convAccum[ch][currentPos]
    //     m_convAccum[ch][currentPos] = 0  // 出力位置をゼロクリアして再利用
    //     currentPos = (currentPos + 1) % (2*blockSize)
};
```

> O(N²) だが最初のごく少数のサンプルに限定されるため許容範囲。レイテンシはゼロ。

### 7.4 ConvolutionEngineFftStage（周波数領域分割畳み込み）

**目的**: 対応する IR ブロック範囲を FFT ベースの Overlap-Add 法で畳み込む。

```cpp
template<uint32_t _blockSize, uint32_t _blockCountMax, bool _replacesDirectStage>
class ConvolutionEngineFftStage
{
    // FFTサイズ（時間域）= 2 * blockSize
    // FFTサイズ（周波数域）= blockSize + 1  （実数→複素ハーフスペクトル）

    // データ配置（alignas(16) 済み）：
    // m_audioInputBuffer[buffers][2][2*blockSize]    // オーディオ入力バッファ
    // m_audioOutputBuffer[buffers][2][blockSize]     // 出力バッファ
    // m_AUDIO_IN_BLOCKS[2][blockCountMax][halfSpec]  // FFT済み過去オーディオの循環バッファ
    // m_ir[2][2][blockCountMax]                       // IR ポインタ
    // m_IR_BLOCK          （ALWAYS_UPDATE_IR_BLOCKS==1時は毎回FFT）
    // m_IR_BLOCKS[2][n][halfSpec]   （キャッシュ時）
    // m_CONV[halfSpec]              // 周波数領域畳み込みアキュムレータ
    // m_conv[2*blockSize]           // IFFT 結果バッファ
    // m_overlap[2][blockSize]      // OLA 用オーバーラップ保存
    // m_fft, m_ifft                 // Fft インスタンス
    // m_thread                      // DSPThread（処理オフロード用）
};
```

**process()（オーディオスレッド側）**：
1. audioIn を `m_audioInputBuffer[writeIdx][ch]` に書き込みながら、`m_audioOutputBuffer[writeIdx][ch]` を audioOut に加算（過去 result を累積）。
2. `audioBufferPtr += audioProcessingBlockSize`。
3. トリガ点 `convProcessingPointSamples` に到達したら：
   - スレッド使用時: `m_audioProcessBufferIndex` にバッファを記憶して `m_thread.notify()`。
   - インライン時: 即座に `convolutionProcessOnSignal()` を実行。
4. `audioBufferPtr == blockSize` でバッファを一圈巡回。

**convolutionProcessOnSignal()（DSPスレッドまたはインライン）**：
```cpp
for each channel ch:
    // 現オーディオブロックをFFT
    m_fft.process(in, m_AUDIO_IN_BLOCKS[ch][writePtr]);

    memset(m_CONV, 0, halfSpec);

    // 全 past ブロックとの畳み込みを累積
    for b in [0, blockCount):
        readPtr = (writePtr - b + blockCount) % blockCount;
        if ALWAYS_UPDATE_IR_BLOCKS:
            fft(ir[ch][b]) -> m_IR_BLOCK;         // 毎回 FFT（メモリ節約）
            m_ifft.convolve_accum(m_CONV, m_IR_BLOCK, m_AUDIO_IN_BLOCKS[ch][readPtr]);
        else:
            m_ifft.convolve_accum(m_CONV, m_IR_BLOCKS[ch][b], m_AUDIO_IN_BLOCKS[ch][readPtr]);

    // IFFT -> 時間領域
    m_ifft.process(m_conv, m_CONV);

    // OLA: 前半 + 前回overlap を出力、後半を次回overlap に保存
    for i in [0, blockSize):
        out[i] = m_conv[i] + m_overlap[ch][i];
    memcpy(m_overlap[ch], &m_conv[blockSize], blockSize);
```

**レイテンシ制御**:
- `_replacesDirectStage == true` の場合：convolution が最新オーディオブロック（H0〜H1）を FFT ステージで処理し、レイテンシ追加なし。
- `_replacesDirectStage == false` の場合：処理開始が `convProcessingPointSamples = (blockSize > audioProcessingBlockSize) ? blockSize : blockSize/2` 遅延。

**IR ホット=swap**：`std::atomic<uint8_t> m_irIndex` を使用。`canUpdateIr()` は `audioBufferPtr == convProcessingPointSamples - audioProcessingBlockSize` のときに true を返す（次回処理の直前の安全な窗口）。

## 8. Fft（Fft.h）— PFFFT ラッパ

```cpp
typedef float cplx_f32[2];  // [Re, Im] インタリーブ

template<bool _isForward, bool _freqDataOrdered>
class Fft {
    PFFFT_Setup* m_setup;        // pffft_new_setup(fftSize, PFFFT_REAL)
    float* m_workData;           // スクラッチバッファ

    void init(uint32_t fftSize, float* workData);
    void exit(void);

    // 順変換 / 逆変換
    void process(float* audioData, cplx_f32* freqBinsData);

    // 周波数領域畳み込み積の累積
    // freqBinsAccum += (freqBinsA * freqBinsB) / fftSize
    void convolve_accum(cplx_f32* freqBinsAccum,
                        cplx_f32* freqBinsDataA,
                        cplx_f32* freqBinsDataB);
};
```

| パス | 使用 API |
|---|---|
| ordered, forward | `pffft_transform_ordered(..., PFFFT_FORWARD)` |
| ordered, backward | `pffft_transform_ordered(..., PFFFT_BACKWARD)` |
| unordered, forward | `pffft_transform(..., PFFFT_FORWARD)` |
| unordered, backward | `pffft_transform(..., PFFFT_BACKWARD)` |
| unordered convolve | `pffft_zconvolve_accumulate(m_setup, A, B, accum, 1.0/N)` |
| ordered convolve | 手動要素ごとの複素乗算 + 割算 |

> ordered は「自然な周波数順序」、unordered は PFFFT 内部のスクランブル済み順序。コードでは `_isForward=false`（逆変換）の `convolve_accum` を呼ぶことで、IFFT と畳み込みを一体化している。

## 9. FilterBiquad（2次バターワース LPF/HPF）

### 9.1 係数設計

双1次変換（bilinear transform） + バターワース正規化。

```cpp
void computeCoefficientsButterworth2ndOrder(double samplerate)
{
    const double omega_c = 2π * m_cutoffFreq_Current / samplerate;
    const double omega   = std::tan(omega_c / 2.0);  // pre-warping
    const double omega2  = omega * omega;
    const double sqrt2_omega = std::sqrt(2.0) * omega;
    const double a0 = 1.0 + sqrt2_omega + omega2;    // 正規化係数

    // a1, a2: LPF/HPF で共通
    a1 = 2.0 * (omega2 - 1.0) / a0;
    a2 = (1.0 - sqrt2_omega + omega2) / a0;

    if (m_lowPass) {        // LPF
        b0 = omega2 / a0;
        b1 = 2.0 * b0;
        b2 = b0;
    } else {                // HPF
        b0 = 1.0 / a0;
        b1 = -2.0 * b0;
        b2 = b0;
    }
}
```

### 9.2 実装トポロジー

**Direct Form II Transposed（DF2T）**：

```cpp
// 状態: m_state[z1], m_state[z2]（double 精度）
// 係数: m_coeffs[b0,b1,b2,a1,a2]（double 精度、16byte align）

acc  = b0 * x;       // y[n] の feedforward 部
acc += d0;            // 状態 z1 を加算
d0   = b1 * x;        // z1 更新: feedforward
d0  -= a1 * acc;      // z1 更新: feedback
d0  += d1;            // z1 更新: cascade from z2
d1   = b2 * x;        // z2 更新: feedforward
d1  -= a2 * acc;      // z2 更新: feedback
output = acc;         // y[n]
```

> 状態・係数共に `alignas(16) double` で宣言。SIMD  векторизация 対応。内部計算は常に `double` だが、テンプレート `_InOutFpType` で I/O は `float` として処理可能。

### 9.3 平滑化

```cpp
void setTargetFreq(float cutoffFreqTarget, float smoothingFactor, double samplerate)
{
    // 平滑化された 목표 周波数を計算
    m_cutoffFreq_Current = DspUtils::expSmoothing(cutoffFreqTarget, m_cutoffFreq_Current, smoothingFactor);
    // 平滑化された周波数で係数再計算
    computeCoefficientsButterworth2ndOrder(samplerate);
}
```

> 平滑化により急激な係数変化（ジッパーノイズ）を防止。

## 10. SamplerateConverter（IR サンプルレート変換）

### 10.1 役割

DAW のサンプルレートが IR の基準（48000 Hz）と異なる場合に IR をリサンプルする。

```cpp
class SamplerateConverter {
public:
    static void convert(double samplerateIn, double samplerateOut,
                        const float* audioIn[2], float* audioOut[2],
                        uint32_t audioInLength, uint32_t audioOutLengthMax,
                        uint32_t& audioOutLengthActual);
};
```

### 10.2 実装

内部で **JUCE ResamplingAudioSource** を使用（`**windowed-sinc FIR**` フィルタ）。

```cpp
void SamplerateConverter::convert(...)
{
    // 出力長計算
    audioOutLengthActual = ceil(audioInLength * (samplerateOut / samplerateIn));
    audioOutLengthActual = min(audioOutLengthActual, audioOutLengthMax);

    for ch in [0, 1]:
        ResamplingAudioSource src(wavData[ch], audioInLength);
        juce::ResamplingAudioSource resampler(&src, false, 1);
        resampler.setResamplingRatio(samplerateIn / samplerateOut);
        resampler.prepareToPlay(audioInLength, samplerateOut);

        juce::AudioBuffer<float> outBuf(&audioOut[ch], 1, audioOutLengthActual);
        juce::AudioSourceChannelInfo info(&outBuf, 0, audioOutLengthActual);
        resampler.getNextAudioBlock(info);  // 1 回で全処理
}
```

> チャンネルごとに独立して JUCE の `ResamplingAudioSource` を生成し処理。出力バッファ最大は `BCNRVRB_IR_MAX_LEN_SAMPLES`（= 3,840,000）。これを超える場合はilent truncation）。

## 11. DspThread（DspThread.h）— シグナル&ウェイト型ワーカースレッド

```cpp
class DspThread : public juce::Thread {
    std::function<void(void)> m_funcInit;       // 1回だけ実行（init）
    std::function<void(void)> m_funcExit;       // 終了時1回（teardown）
    std::function<void(void)> m_funcProcessOnSignal; // 通知毎（convolution処理）
    std::atomic<bool> m_initialized;              // lock-free 初期化完了フラグ

    void run(void) override {
        m_initialized = false;
        m_funcInit();           // 初期化
        m_initialized = true;

        while (!threadShouldExit()) {
            if (wait(-1.0)) {   // 無限待機（notify() で起床）
                if (threadShouldExit()) break;
                m_funcProcessOnSignal();  // 畳み込み処理
            }
        }
        m_funcExit();           // 終了処理
        m_initialized = false;
    }
};
```

- `ConvolutionReverb` 本体の IR 更新用スレッド（`updateIr()`）に登録。
- 各 `ConvolutionEngineFftStage` のうち、`blockSize > audioProcessingBlockSize` の大きいステージのみが処理オフロード用它として使用。
- プラットフォーム別の起動優先度：macOS は `withPriority(8)` または RealtimeOptions、Linux は `Priority::highest`、Windows は `startThread(Priority::high)`。

### 12.2 process() の処理フロー

```
process(audioIn, audioOut, stereo, samplerate, blockSize,
         decayControl, colorControl, dryWetControl, irIndex)
│
├─ 整合性チェック（blockSize, samplerate）
│   └─ 不適合時: dry pass-through でリターン
│
├─ パラメータ変化検出（IR, SR, block, ch）
│   └─ 変化時: reconfigure() を実行
│
├─ dry/wet ramp 生成
│   ├─ dryTarget = LUT[dryWetControl < 0 ? 1.0 : 1 - dryWetControl]
│   ├─ wetTarget = LUT[dryWetControl > 0 ? 1.0 : 1 + dryWetControl]
│   └─ smoothParameter() で sample-level ramp を生成し
│      audioDry[ch][i] = in[ch][i] * dry
│      audioReverbIn[ch][i] = in[ch][i] * wet
│
├─ IR 切替チェック（canUpdateIr && !updatingIr）
│   ├─ m_convolutionEngine.updateIr(m_irUpdateIndex)
│   ├─ m_irUpdateIndex をトグル
│   ├─ m_decayControl = decayControl (atomic)
│   ├─ m_colorControl = colorControl (atomic)
│   ├─ m_updatingIr = true
│   └─ m_thread.notify() → updateIr() を別スレッドで起動
│
├─ 畳み込み実行
│   └─ m_convolutionEngine.process(audioReverbIn, audioOut)
│
├─ dry 混合: audioOut[ch][i] += audioDry[ch][i]
│
└─ smooth 状態を保存
```

### 12.3 reconfigure()（IR/SR/block 変更時に1回実行）

```
1. m_thread を再起動（プラットフォーム別の優先度設定）
2. dry/wet平滑係数を再計算（m_dryWetSmoothingFactor, m_dryWetRecalculateTimesPerBlock）
3. m_convolutionEngine.exit()
4. IR ロード:
   - SR 一致時: memcpy
   - SR 不一致時: SamplerateConverter::convert でリサンプル
5. IR 正規化（post-size, pre-color）
   - RMS 計算 → factor = 0.65 / sqrt(sumSquares)
   - 全サンプルに乗算
6. m_convolutionEngine.init(irPreProcessed, ...) ← ポスト処理前を渡す
7. フィルタを reset()
```

### 12.4 updateIr()（バックグラウンドスレッド）

```
1. decay 処理:
   - LUT から decayTarget を取得
   - expSmoothing で平滑化（m_decayCurrent）
   - カット点: irLen * m_decayCurrent
   - カット点以降を expSmoothing(target=0) でテーパダウン
   - 各サンプルに乗算
2. color 処理:
   - colorControl <= 0 → LPF モード、> 0 → HPF モード
   - Fc を LUT から取得（LUT は指数曲線）
   - LPF/HPF とも有効に（LPF: fc=20kHz、HPF: fc=20Hz を相手側に設定）
   - 各 ch に対して clearState → setTargetFreq → process(ir, ir, irLen)
3. m_updatingIr = false
```

## 14. 処理パイプライン（全文流れ）

```
DAW processBlock()
│
├─ BarcelonaReverberaAudioProcessor::processBlock()
│   │
│   ├─ m_convolutionReverb.process()
│   │   │
│   │   ├─ dry/wet ramp 生成（smoothParameter, 5ms 時定数）
│   │   │
│   │   ├─ canUpdateIr && !updatingIr → updateIr() 起動
│   │   │   └─ DspThread::run() → updateIr()
│   │   │       ├─ decay envelope 適用
│   │   │       ├─ LPF + HPF 処理（in-place）
│   │   │       └─ m_updatingIr = false
│   │   │
│   │   ├─ ConvolutionEngine::process()
│   │   │   │
│   │   │   ├─ audioOut ← 0
│   │   │   │
│   │   │   ├─ DirectStage::process()   // H0-H1: O(N²), レイテンシ0
│   │   │   │   └─ audioOut += timeDomainConv(dry, IR0..IR1)
│   │   │   │
│   │   │   ├─ FftStage<BS>::process()      // H2-H3: FFT 64
│   │   │   ├─ FftStage<2BS>::process()     // H4-H7: FFT 128
│   │   │   ├─ FftStage<4BS>::process()     // H8-H15: FFT 256
│   │   │   ├─ ...
│   │   │   └─ FftStage<LONGEST>::process() // H[ остальное ]: FFT 16384
│   │   │       各 stage ごと:
│   │   │         - オーディオバッファ蓄積
│   │   │         - トリガ点で FFT → convolve → IFFT → OLA → audioOut に加算
│   │   │         - （大きい stage は DSPThread で処理）
│   │   │
│   │   └─ audioOut += audioDry  // dry 混合
│   │
│   └─ buffer に outputData をコピー
│
└─ DAW へ返送
```

## 15. スレッドモデルまとめ

| スレッド | 生成元 | 役割 | 優先度 |
|---|---|---|---|
| オーディオコールバックスレッド | DAW/VST3 Host | `processBlock` → `m_convolutionReverb.process` | 最高（Real-time） |
| `ConvolutionReverb` 更新スレッド | `reconfigure()` | `updateIr()`（decay/color 処理） | 高（通常スレッド） |
| `FftStage` DSP スレッド | `FftStage::init()` | 大きい FFT ステージの畳み込み処理 | 高（OS 次第） |

> スレッド間同期は `std::atomic` のみ（ロックなし）。IR ダブルバッファ (`irPostProcessed[2]`) とオーディオダブルバッファ (`m_audioInputBuffer[2]`) によりロックフリーなデータ交換を実現。

## 17. 設計上のハイライト

| # | ポイント | 詳細 |
|---|---|---|
| 1 | **非一様分割畳み込み** | ブロックサイズを段階的に倍増（64→128→256→...→16384）し、レイテンシ最小化と計算量削減の両立 |
| 2 | **Direct Stage による零レイテンシ** | IR 最初の 2 ブロックを時間領域直接畳み込み。最短レイテンシを実現 |
| 3 | **コンパイル時ステージ生成** | テンプレートメタプログラミングにより DSP ディスパッチの 런타임オーバーヘッドを完全排除 |
| 4 | **ロックフリー IR 切替** | `std::atomic<uint8_t>` によるダブルバッファ。原子的 swap でオーディオを止めずに IR 切替 |
| 5 | **DSPThread による処理分離** | 重い FFT をオーディオスレッドから切り離しドロップアウトを防止 |
| 6 | **PFFFT** | SIMD ベクトル化（AVX/NEON）された Split-radix 実数 FFT |
| 7 | **Audible IR 処理** | 正規化（0.65 / RMS）→ decay envelope → LPF/HPF の順でポスト処理 |
| 8 | **パラメータ平滑化** | 80ms 時定数の指数平滑でノブ操作時の click/pop を防止 |
| 9 | **DF2T バターワース** | 内部 double 精度、16byte アライメント、SIMD 対応 |
| 10 | **リサンプル** | JUCE ResamplingAudioSource（windowed-sinc FIR）による高品质変換 |
| 11 | **ビルド Generators** | `ConvertWavsToCArray.py` で WAV → C++ ヘッダ変換の自动化 |
| 12 | **VST3 + AU 対応** | JUCE によるクロスプラットフォーム抽象化で単一コードベースから両形式を生成 |
