### E1.3 SoftClipOS遅延実測手順【確定】

1. 単位インパルス `[1.0, 0, 0, ...]` 64サンプルを生成
2. `softClipOS.processUp()` で2倍にアップサンプル（128サンプル）
3. 何もせず `softClipOS.processDown()` でダウンサンプル
4. 出力のピーク位置 - 入力のピーク位置 = 実測群遅延
5. この値を `softClipLatencyBaseRateSamples` に設定

## S1. 音響工学文献調査による確定事項（2026-06-22）

### S1.1 線形位相FIRの群遅延公式【文献確定】

**調査内容**: Halfband FIRフィルタの群遅延推定式 `(taps-1)/2` の妥当性

**文献**: DSP関連公式文献（dsprelated.com, dsp.stackexchange.com, MathWorks documentation）:

- 対称タップFIR（type I/II）の群遅延は `(N-1)/2` サンプルが**数学的に厳密な公式**
  - 文献: dsp.stackexchange.com "Group delay of the FIR filter" : "For an FIR filter, with symmetrical tap values, the group delay is (N-1)/2"
- 31tap halfband → `(31-1)/2 = 15 samples @ 2x rate`
- base rate換算: `15 / 2 = 7.5 samples`
- processUp: 15 samples @ 2x = 7.5 base rate
- processDown: 15 samples @ 2x = 7.5 base rate
- **合計: 15 base rate samples = 0.3125ms @ 48kHz**

**確定**: 理論式 `(taps-1)/2` は**線形位相FIRの厳密な（近似ではない）群遅延公式**。
ポリフェーズ実装（centerTap, centerDelayInput, convParity）は畳み込み演算の実装詳細に過ぎず、
フィルタの線形位相特性と群遅延に影響しない。理論値15 base rate samplesは確定値。
実装後の実測確認は「検証」として推奨するが、補正は不要。

### S1.2 TruePeak補間tap数【文献確定】

**調査内容**: BS.1770-4/5 準拠TruePeak測定の補間FIR長

**文献**:

- ITU-R BS.1770-3/4/5, essentia TruePeakDetector
- Thomas Holm Hansen (2012) "True-peak interpolation filter comparison ITU-R BS.1770-3 example vs DK-Technologies"

### S1.4 LUFS Integrated — ゲーティングアルゴリズム【文献確定】

**調査内容**: Integrated Loudness (LUFS) の2段ゲーティング仕様

**文献**: ITU-R BS.1770-4 §4.1, EBU Tech 3341 (2011), EBU R128 (2020):

- **ブロック長**: Momentary = 400ms (矩形), Short-term = 3000ms (矩形, 75%オーバーラップ)
- **Absolute gate**: -70 LUFS (LKFS) — 絶対無音ゲート
- **Relative gate**: Absolute-gated loudness - 10 LU — (-10dB相対ゲート、EBU R128)
- **Integrated (Programme Loudness)**: 2-pass gating:
  1. パス1: Absolute gate (-70 LUFS) で全ブロックをフィルタ → 平均値 L_abs を算出
  2. パス2: Relative gate threshold = L_abs - 10 LU で再度フィルタ → 最終的な Integrated Loudness L_Int を算出
- **チャンネル重み**: L=1.0, R=1.0, C=1.0, Ls=1.41, Rs=1.41 (5ch時)

**確定**: 計画書の「2段（gating）」記述は正しい。詳細: 絶対ゲート(−70 LUFS)→相対ゲート(−10 LU below absolute-gated)の2パス。Momentary(400ms), Short-term(3s)の時間窓も確定。

### S1.5 TruePeak Limiter Lookahead【文献確定】

**調査内容**: TruePeakリミッターのルックアヘッド時間

**文献**: マスタリング業界標準（FLUX::, Waves, Universal Audio, FabFilter Pro-L）:

- **標準的lookahead範囲**: 0.5〜5ms
- **透明性重視（マスタリング）**: 0.5〜2ms（過渡特性を維持）
- **安全策重視（放送/配信）**: 2〜5ms（全TruePeakを確実に捕捉）
- ConvoPeqのTruePeak検出が4x oversampleベースのため、補間による検出遅延 + lookaheadで最大歪を補償する必要

**確定**: 5msは安全側の設計として妥当。特にConvoPeqがリアルタイム分散処理（ISR Bridge Runtime）で動作することを考慮すると、処理ジッタ吸収も含めた5msは適切。**ただし、設定可能（0.5〜5ms）にしておくことが望ましい**。

### S1.6 ADAA — 学術的根拠【文献確定】

**調査内容**: Antiderivative Antialiasing (ADAA) の理論的正当性

**文献**:

1. Parker, J.D., Esqueda, F., & Zölzer, U. (2019) "Antiderivative Antialiasing for Stateful Systems" — DAFx-19, Birmingham, UK
2. Werner, K.J. (2020) "An Equivalent Circuit Interpretation of Antiderivative Antialiasing" — DAFx20in21, Vienna, Austria
3. Gabrielli, L. (2025) "Simplifying Antiderivative Antialiasing with Lookup Table Integration" — DAFx25, Ancona, Italy

**ADAAの基本公式**:

- メモリレス非線形関数 f(x) に対して、差分商近似:
  `y[n] = (F(x[n]) - F(x[n-1])) / (x[n] - x[n-1])`
  ただし F(x) = ∫f(x)dx
- 一次のオーバーサンプリング等価として機能し、折り返し歪を約20dB低減
- Padé近似 f(x) = x(27+x²)/(27+9x²) の場合、F(x) = x²/18 + (4/3)·ln(x²+3) が不定積分の閉形式

**確定**: 設計書のADAA式 `F(x)=x²/18+(4/3)·ln(x²+3)` は Parker/Esqueda (2019) の理論に基づく正しい導出。現行設計で問題なし。

### S1.8 musicalSoftClipScalar — smoothstep+tanh設計【文献検証】

**調査内容**: `musicalSoftClipScalar()` のアルゴリズム設計（smoothstepブレンド + tanh近似 + asymmetry）

**文献による分析**:

- **smoothstep (Hermite補間)**: `t²(3-2t)` — 線形→ソフトクリップ間のクロスフェードに使用。CG業界の標準的スムーズ遷移関数
- **fastTanh Padé(3,2)近似**: `x*(27+x²)/(27+9x²)` — Vicanek (2015) の数値安定tanh近似。"Fast and Stable Approximation of tanh" で提案
- **Kneeブレンド**: `linear*(1-ks) + clipped*ks` — threshold/knee区間での自然な遷移
- **Asymmetry**: 非対称クリッピング（プッシュプル非対称歪）
