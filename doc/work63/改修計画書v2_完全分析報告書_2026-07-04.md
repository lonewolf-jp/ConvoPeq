## 11. 改修計画書 v2 に対する追加提言

### 11.1 DiagnosticsConfig.h の管理

`DiagnosticsConfig.h` が `#define CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS 1` でオーバーライドしているため、**CMakeの設定が実質無視**されている。

```
CMakeLists.txt: option(OFF) → DiagnosticsConfig.h: #define 1 → 常に有効
```

**推奨：**
1. リリースビルドでは `#define CONVOPEQ_ENABLE_RUNTIME_DIAGNOSTICS 0` に変更（診断コードのバイナリサイズ削減）
2. または CMakeLists.txt の option を ON で管理し、header override はデバッグ用途として明示

### 11.2 MMCSS適用の確認方法

`[MMCSS]` ログが今回も含めて全4セッションで出力されていない。次回は以下を確認：
1. `useMmcssPriority` の設定値をログ先頭で出力
2. または `[DIAG] prepareToPlay:` の下で MMCSS 状態を明示

### 11.3 cbD常時発生の解消

`cbD=3/tick` は MaxDrainPerTick=16 で約16%のCB_ARRIVALがdropされていることを示す。

```
cbW(書込) = 16/tick → 現状のMaxDrainPerTick(16) と同数
cbD(drop) = 3/tick  → 他のDiagEvent(CPU_MIG等)が枠を消費
```

**推奨:** MaxDrainPerTick を **16→32** に増加（計画書 v2 の推奨通り）。512 bufferの枯渇リスクは低下（`approxOcc=496` → 30程度に改善見込み）。
